### 1.3 `DROP` y `TRUNCATE` — Eliminar Objetos o Contenido

  

#### ⚙️ Mecánica de la Query

  

Son operaciones opuestas en impacto:

  

```

DROP TABLE:

  ──► Elimina los DATOS de todas las filas

  ──► Elimina la ESTRUCTURA de la tabla

  ──► Elimina los METADATOS del catálogo

  ──► Elimina los ÍNDICES y RESTRICCIONES asociados

  ──► Operación IRREVERSIBLE ⚠️

  

TRUNCATE TABLE:

  ──► Elimina todos los DATOS (reset de auto_increment)

  ──► CONSERVA la estructura, índices y restricciones

  ──► Más rápido que DELETE sin WHERE (no registra cada fila)

  ──► En la mayoría de motores: NO puede deshacerse con ROLLBACK ⚠️

  

DELETE sin WHERE (para comparar):

  ──► Elimina fila por fila (registra cada operación en el log)

  ──► CONSERVA estructura

  ──► SÍ puede deshacerse con ROLLBACK ✅

```

  

#### 🔤 Sintaxis Estándar

  

```sql

-- Eliminar tabla completa (estructura + datos)

DROP TABLE [IF EXISTS] nombre_tabla;

  

-- Eliminar base de datos completa

DROP DATABASE [IF EXISTS] nombre_bd;

  

-- Vaciar contenido, conservar estructura

TRUNCATE TABLE nombre_tabla;

  

-- Renombrar tabla

RENAME TABLE nombre_viejo TO nombre_nuevo;

```

  

#### 🗄️ Categoría Estricta

**DDL** — `DROP` y `TRUNCATE` operan sobre la estructura o la resetean completamente.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 3: DROP, TRUNCATE, RENAME

-- ══════════════════════════════════════════════════════

  

-- Crear tabla temporal para el demo

CREATE TABLE logs_temporales (

    id          INT AUTO_INCREMENT PRIMARY KEY,

    mensaje     VARCHAR(255),

    fecha       TIMESTAMP DEFAULT CURRENT_TIMESTAMP

);

  

-- Insertar datos de prueba

INSERT INTO logs_temporales (mensaje) VALUES

    ('Error 404 en /productos'),

    ('Login exitoso usuario #42'),

    ('Timeout en consulta lenta');

  

-- Ver los datos antes de truncar

SELECT * FROM logs_temporales;

-- Resultado: 3 filas con sus ids 1, 2, 3

  

-- TRUNCATE: vacía la tabla y resetea el AUTO_INCREMENT

TRUNCATE TABLE logs_temporales;

  

-- La tabla existe pero está vacía

SELECT * FROM logs_temporales;     -- 0 filas

SELECT COUNT(*) FROM logs_temporales; -- 0

  

-- Si insertamos de nuevo, el id empieza desde 1 (reseteo)

INSERT INTO logs_temporales (mensaje) VALUES ('Nuevo inicio');

SELECT * FROM logs_temporales;     -- id = 1 (no id = 4)

  

-- Renombrar la tabla

RENAME TABLE logs_temporales TO auditoria_sistema;

  

-- Eliminar la tabla renombrada (IF EXISTS evita error si no existe)

DROP TABLE IF EXISTS auditoria_sistema;

  

-- Verificar que ya no existe

SHOW TABLES LIKE 'auditoria%';    -- resultado vacío ✅

```
