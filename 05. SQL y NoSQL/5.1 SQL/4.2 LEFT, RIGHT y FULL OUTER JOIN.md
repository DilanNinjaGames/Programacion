### 4.2 LEFT, RIGHT y FULL OUTER JOIN

#### ⚙️ Mecánica de la Query

  

```

LEFT JOIN:

  Tabla A (izquierda) ──────── Tabla B (derecha)

  ████████████████████ ◄──── todos los de A

                       ────► solo los que hacen match en B

                       ────► si no hay match: columnas de B = NULL

  

  resultado = TODOS los de A + los que hacen match en B

  

RIGHT JOIN: (imagen espejo del LEFT JOIN)

  resultado = los que hacen match en A + TODOS los de B

  

FULL OUTER JOIN:

  resultado = TODOS de A + TODOS de B (NULLs donde no hay match)

  (MySQL no lo soporta nativamente → simular con LEFT UNION RIGHT)

```

  

#### 🔤 Sintaxis Estándar

  

```sql

-- LEFT JOIN: todos de la izquierda + matches de la derecha

SELECT *

FROM   tabla_izquierda  t_izq

LEFT JOIN tabla_derecha t_der ON t_izq.col = t_der.col;

  

-- RIGHT JOIN: todos de la derecha + matches de la izquierda

SELECT *

FROM   tabla_izquierda  t_izq

RIGHT JOIN tabla_derecha t_der ON t_izq.col = t_der.col;

  

-- FULL OUTER JOIN (simulado en MySQL con UNION)

SELECT * FROM tabla_izq

LEFT JOIN  tabla_der ON tabla_izq.col = tabla_der.col

UNION

SELECT * FROM tabla_izq

RIGHT JOIN tabla_der ON tabla_izq.col = tabla_der.col;

```

  

#### 🗄️ Categoría Estricta

**DQL** — Cláusula de consulta.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 9: OUTER JOINs — Detectar huérfanos y vacíos

-- ══════════════════════════════════════════════════════

  

-- LEFT JOIN: todas las categorías, incluso las sin productos

SELECT

    c.nombre        AS categoria,

    c.activa,

    COUNT(p.id_producto) AS total_productos   -- 0 para categorías vacías

FROM categorias c                              -- tabla IZQUIERDA: todas las categorías

LEFT JOIN productos p                          -- tabla derecha: solo si hay match

    ON c.id_categoria = p.id_categoria

GROUP BY c.id_categoria, c.nombre, c.activa

ORDER BY total_productos DESC;

-- Resultado: muestra TODAS las categorías, incluso "Descontinuados" con 0 productos

  

-- Detectar categorías SIN productos (las que tienen NULL en la parte derecha)

SELECT

    c.nombre    AS categoria_sin_productos

FROM categorias c

LEFT JOIN productos p ON c.id_categoria = p.id_categoria

WHERE p.id_producto IS NULL;    -- ← truco: si la FK es NULL, no hubo match → sin productos

  

-- Detectar productos SIN categoría asignada (NULL en FK)

SELECT

    p.nombre    AS producto_sin_categoria,

    p.precio

FROM productos p

LEFT JOIN categorias c ON p.id_categoria = c.id_categoria

WHERE c.id_categoria IS NULL;   -- ← producto huérfano (no tiene categoría)

  

-- FULL OUTER JOIN simulado en MySQL:

-- Obtener TODAS las categorías Y todos los productos, independientemente del match

SELECT c.nombre AS categoria, p.nombre AS producto

FROM   categorias c

LEFT JOIN  productos p ON c.id_categoria = p.id_categoria

UNION

SELECT c.nombre AS categoria, p.nombre AS producto

FROM   categorias c

RIGHT JOIN productos p ON c.id_categoria = p.id_categoria;

```
