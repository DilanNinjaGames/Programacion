### 3.1 Anatomía Completa de un `SELECT`

#### ⚙️ Mecánica de la Query

El motor **NO ejecuta las cláusulas en el orden en que se escriben**. El orden real de ejecución es:

```

Orden de ESCRITURA:     Orden de EJECUCIÓN:

─────────────────       ─────────────────────────────────────

1. SELECT           →   1. FROM      (identifica las tablas)

2. FROM             →   2. JOIN      (cruza tablas)

3. WHERE            →   3. WHERE     (filtra filas individuales)

4. GROUP BY         →   4. GROUP BY  (agrupa filas)

5. HAVING           →   5. HAVING    (filtra grupos)

6. ORDER BY         →   6. SELECT    (proyecta columnas)

7. LIMIT            →   7. DISTINCT  (elimina duplicados)

                    →   8. ORDER BY  (ordena el resultado)

                    →   9. LIMIT     (recorta filas finales)

```

  

**Implicación práctica:** No puedes usar alias del `SELECT` en el `WHERE` (el WHERE se ejecuta antes del SELECT). Sí puedes usarlos en `ORDER BY` y `HAVING` en algunos motores.

  

#### 🔤 Sintaxis Estándar

  

```sql

SELECT   [DISTINCT] columna1, columna2, expresion AS alias

FROM     tabla [AS alias_tabla]

[JOIN    otra_tabla ON condicion]

WHERE    condicion_de_fila

GROUP BY columna_agrupadora

HAVING   condicion_de_grupo

ORDER BY columna [ASC | DESC]

LIMIT    cantidad [OFFSET inicio];

```

  

#### 🗄️ Categoría Estricta

**DQL** (subconjunto de DML en muchas clasificaciones) — Solo lee, nunca modifica datos.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 7: SELECT completo con todos sus componentes

-- ══════════════════════════════════════════════════════

  

-- Consulta simple: todas las columnas de una tabla

SELECT * FROM categorias;

  

-- Proyección: solo las columnas que necesitamos

SELECT nombre, precio, stock

FROM   productos;

  

-- Con alias de columna y tabla

SELECT

    p.nombre        AS producto,            -- alias de columna con AS

    p.precio        AS precio_base,

    p.precio * 0.90 AS precio_con_descuento -- expresión calculada en SELECT

FROM productos AS p;                        -- alias de tabla

  

-- DISTINCT: eliminar duplicados en el resultado

SELECT DISTINCT id_categoria

FROM   productos;                           -- solo categorías que tienen productos

  

-- Filtros con WHERE y múltiples condiciones

SELECT nombre, precio, stock

FROM   productos

WHERE  precio > 50                          -- precio mayor a 50

  AND  stock > 100                          -- Y con stock suficiente

  AND  id_categoria IN (1, 4)              -- Y de Electrónica o Deportes

ORDER BY precio DESC;                       -- ordenar de mayor a menor precio

  

-- BETWEEN: rango inclusivo de precios

SELECT nombre, precio

FROM   productos

WHERE  precio BETWEEN 20.00 AND 100.00     -- incluye los extremos 20 y 100

ORDER BY precio ASC;

  

-- LIKE: búsqueda por patrón de texto

SELECT nombre

FROM   productos

WHERE  nombre LIKE '%Pro%';                -- contiene 'Pro' en cualquier posición

-- % = cero o más caracteres, _ = exactamente un carácter

  

SELECT nombre

FROM   categorias

WHERE  nombre LIKE 'E%';                   -- empieza con 'E' → Electrónica

  

-- IS NULL / IS NOT NULL

SELECT nombre, id_categoria

FROM   productos

WHERE  id_categoria IS NOT NULL;           -- solo productos con categoría asignada

  

-- LIMIT con OFFSET (paginación)

SELECT nombre, precio

FROM   productos

ORDER BY precio DESC

LIMIT  3 OFFSET 0;                         -- página 1: primeros 3 resultados

  

SELECT nombre, precio

FROM   productos

ORDER BY precio DESC

LIMIT  3 OFFSET 3;                         -- página 2: siguientes 3 resultados

```
