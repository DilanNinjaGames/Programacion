### 6.1 `GRANT`, `REVOKE` y `DENY`

#### ⚙️ Mecánica de la Query

```

Sistema de permisos en capas:

  

Usuario ──► tiene ROL ──► que tiene PRIVILEGIOS ──► sobre OBJETOS

  

GRANT:  agrega un permiso en la tabla de control de acceso

REVOKE: elimina un permiso previamente concedido

DENY:   prohíbe explícitamente (tiene precedencia sobre GRANT en SQL Server)

  

Privilegios comunes:

SELECT, INSERT, UPDATE, DELETE  → sobre tablas o vistas

EXECUTE                         → sobre procedimientos almacenados

ALL PRIVILEGES                  → todos los permisos

CREATE, DROP, ALTER             → permisos DDL

```

  

#### 🔤 Sintaxis Estándar

  

```sql

-- Conceder permisos a un usuario

GRANT SELECT, INSERT ON base_datos.tabla TO 'usuario'@'host';

GRANT ALL PRIVILEGES  ON base_datos.*    TO 'usuario'@'host';

  

-- Conceder con posibilidad de que el usuario propague el permiso

GRANT SELECT ON tabla TO 'usuario'@'host' WITH GRANT OPTION;

  

-- Retirar permisos

REVOKE INSERT ON base_datos.tabla FROM 'usuario'@'host';

  

-- Ver los permisos actuales de un usuario

SHOW GRANTS FOR 'usuario'@'host';

```

  

#### 🗄️ Categoría Estricta

**DCL** — Gestiona permisos y control de acceso, no datos ni estructura.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 13: DCL — Gestión de permisos (MySQL)

-- ══════════════════════════════════════════════════════

  

-- Crear usuario de solo lectura (para reportes)

CREATE USER IF NOT EXISTS 'analista_reportes'@'localhost'

IDENTIFIED BY 'password_seguro_123';

  

-- Conceder solo SELECT sobre la base de datos de la tienda

GRANT SELECT ON tienda_online.* TO 'analista_reportes'@'localhost';

  

-- Crear usuario para la aplicación web (necesita leer y escribir, no borrar)

CREATE USER IF NOT EXISTS 'app_web'@'%'  -- % = desde cualquier host

IDENTIFIED BY 'otro_password_456';

  

GRANT SELECT, INSERT, UPDATE

    ON tienda_online.productos

    TO 'app_web'@'%';

  

GRANT SELECT, INSERT, UPDATE

    ON tienda_online.ventas

    TO 'app_web'@'%';

  

-- Verificar los permisos concedidos

SHOW GRANTS FOR 'analista_reportes'@'localhost';

SHOW GRANTS FOR 'app_web'@'%';

  

-- Revocar el permiso de UPDATE que ya no es necesario para app_web en productos

REVOKE UPDATE ON tienda_online.productos FROM 'app_web'@'%';

  

-- Verificar el cambio

SHOW GRANTS FOR 'app_web'@'%';

  

-- En producción: aplicar los cambios de privilegios inmediatamente

FLUSH PRIVILEGES;

  

-- Eliminar usuario (ej: empleado que dejó la empresa)

DROP USER IF EXISTS 'analista_reportes'@'localhost';

```
