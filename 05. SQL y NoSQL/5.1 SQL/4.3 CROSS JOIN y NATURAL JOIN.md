### 4.3 CROSS JOIN y NATURAL JOIN

#### ⚙️ Mecánica de la Query

  

```

CROSS JOIN (Producto Cartesiano):

  Tabla A (3 filas) × Tabla B (4 filas) = 12 filas resultado

  Cada fila de A se combina con CADA fila de B

  Sin condición ON — simplemente todas las combinaciones posibles

  ⚠️ Con tablas grandes puede generar millones de filas

  

NATURAL JOIN:

  Une automáticamente por columnas de MISMO NOMBRE en ambas tablas

  Equivale a INNER JOIN con ON tabla1.col = tabla2.col

  para cada columna con nombre compartido

  ⚠️ Peligroso: si agregas columnas con el mismo nombre, el JOIN cambia silenciosamente

```

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 10: CROSS JOIN y NATURAL JOIN

-- ══════════════════════════════════════════════════════

  

-- CROSS JOIN: generar todas las combinaciones de colores y tallas

CREATE TABLE colores (color VARCHAR(20));

CREATE TABLE tallas  (talla VARCHAR(5));

  

INSERT INTO colores VALUES ('Rojo'), ('Azul'), ('Negro');

INSERT INTO tallas  VALUES ('S'), ('M'), ('L'), ('XL');

  

-- CROSS JOIN: 3 colores × 4 tallas = 12 combinaciones

SELECT

    c.color,

    t.talla,

    CONCAT(c.color, '-', t.talla) AS variante  -- combinación como código de SKU

FROM colores c

CROSS JOIN tallas t                             -- sin condición ON

ORDER BY c.color, t.talla;

-- Resultado: Azul-L, Azul-M, Azul-S, Azul-XL, Negro-L... (12 filas)

  

-- NATURAL JOIN: une automáticamente por columnas de igual nombre

-- (Ejemplo didáctico — en producción preferir INNER JOIN explícito)

CREATE TABLE empleados_dept (

    id_empleado INT,

    nombre      VARCHAR(100),

    id_departamento INT

);

  

CREATE TABLE departamentos (

    id_departamento INT PRIMARY KEY,  -- mismo nombre que en empleados_dept

    nombre_dept     VARCHAR(100)

);

  

INSERT INTO empleados_dept VALUES (1, 'Ana', 10), (2, 'Luis', 20), (3, 'Mia', 10);

INSERT INTO departamentos  VALUES (10, 'Tecnología'), (20, 'Ventas'), (30, 'RRHH');

  

-- NATURAL JOIN: une por 'id_departamento' (nombre común en ambas tablas)

SELECT e.nombre, d.nombre_dept

FROM   empleados_dept e

NATURAL JOIN departamentos d;

-- Equivale a: INNER JOIN departamentos d ON e.id_departamento = d.id_departamento

  

-- Limpieza del lab

DROP TABLE IF EXISTS colores, tallas, empleados_dept, departamentos;

```
