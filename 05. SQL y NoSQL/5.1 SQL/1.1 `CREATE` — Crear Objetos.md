### 1.1 `CREATE` — Crear Objetos

  

#### ⚙️ Mecánica de la Query

  

`CREATE` instrúye al motor a **reservar espacio en el esquema del sistema** para un nuevo objeto (tabla, índice, vista). Al crear una tabla, el motor registra los metadatos en el catálogo interno (`information_schema`): nombre de columnas, tipos, restricciones y relaciones. No toca datos — solo define la **estructura**.

  

```

Motor SQL:

CREATE TABLE ──► Valida sintaxis

                 ──► Verifica que no existe

                     ──► Registra metadatos en catálogo

                         ──► Reserva estructura en disco

                             ──► Tabla vacía disponible ✅

```

  

#### 🔤 Sintaxis Estándar

  

```sql

-- Base de datos

CREATE DATABASE nombre_bd;

  

-- Tabla con restricciones

CREATE TABLE nombre_tabla (

    columna1  TIPO_DATO   RESTRICCIONES,

    columna2  TIPO_DATO   RESTRICCIONES,

    CONSTRAINT pk_nombre  PRIMARY KEY (columna1),

    CONSTRAINT fk_nombre  FOREIGN KEY (columna2) REFERENCES otra_tabla(col)

);

  

-- Vista

CREATE VIEW nombre_vista AS

    SELECT ...;

  

-- Índice (acelera búsquedas)

CREATE INDEX nombre_indice ON tabla(columna);

```

  

#### 🗄️ Categoría Estricta

**DDL** — Define estructura, no manipula datos. No puede deshacerse con ROLLBACK en la mayoría de motores.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 1: CREATE — Sistema de Tienda Online

-- ══════════════════════════════════════════════════════

  

-- Crear la base de datos del proyecto

CREATE DATABASE IF NOT EXISTS tienda_online;

USE tienda_online;

  

-- Tabla PADRE: Categorías de productos

CREATE TABLE categorias (

    id_categoria  INT           NOT NULL AUTO_INCREMENT,  -- PK autoincremental

    nombre        VARCHAR(100)  NOT NULL,                 -- nombre obligatorio

    descripcion   TEXT,                                   -- descripción opcional

    activa        BOOLEAN       DEFAULT TRUE,             -- activa por defecto

    CONSTRAINT pk_categorias PRIMARY KEY (id_categoria)  -- restricción de PK nombrada

);

  

-- Tabla HIJA: Productos (referencia a categorías)

CREATE TABLE productos (

    id_producto   INT             NOT NULL AUTO_INCREMENT,

    nombre        VARCHAR(200)    NOT NULL,

    precio        DECIMAL(10, 2)  NOT NULL CHECK (precio >= 0),  -- precio no negativo

    stock         INT             DEFAULT 0,

    id_categoria  INT,                                    -- FK hacia categorias

    fecha_alta    DATE            DEFAULT (CURRENT_DATE),

    CONSTRAINT pk_productos  PRIMARY KEY (id_producto),

    CONSTRAINT fk_categoria  FOREIGN KEY (id_categoria)

                             REFERENCES categorias(id_categoria)

                             ON DELETE SET NULL           -- si se borra la categoría, FK = NULL

);

  

-- Vista: productos activos con su categoría (encapsula la JOIN)

CREATE VIEW vista_catalogo AS

    SELECT

        p.nombre        AS producto,

        p.precio,

        p.stock,

        c.nombre        AS categoria

    FROM productos p

    INNER JOIN categorias c ON p.id_categoria = c.id_categoria

    WHERE c.activa = TRUE;

  

-- Índice en precio para acelerar búsquedas por rango de precio

CREATE INDEX idx_precio ON productos(precio);

```
