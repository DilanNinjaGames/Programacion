### 2.3 `DELETE` — Eliminar Filas

#### ⚙️ Mecánica de la Query

  

```

DELETE ──► Evalúa el WHERE para identificar filas objetivo

          ──► Verifica restricciones de FK (¿alguna tabla hija referencia esta fila?)

              ──► Si hay FK activa: depende de ON DELETE (RESTRICT / CASCADE / SET NULL)

                  ──► Elimina fila por fila (registra cada una en el log)

                      ──► Actualiza índices

                          ──► Registra en log → puede deshacerse con ROLLBACK ✅

```

  

**DELETE vs TRUNCATE:**

  

| | `DELETE` sin WHERE | `TRUNCATE` |

|---|---|---|

| Velocidad | Lento (fila por fila) | Muy rápido |

| ROLLBACK | ✅ Sí | ❌ No (generalmente) |

| Resetea AUTO_INCREMENT | ❌ No | ✅ Sí |

| Activa triggers | ✅ Sí | ❌ No |

  

#### 🔤 Sintaxis Estándar

  

```sql

-- Eliminar filas específicas (siempre con WHERE)

DELETE FROM tabla

WHERE  condicion;

  

-- Eliminar con subconsulta

DELETE FROM tabla

WHERE  columna IN (SELECT col FROM otra_tabla WHERE condicion);

  

-- Eliminar con JOIN (MySQL)

DELETE t1

FROM   tabla1 t1

INNER JOIN tabla2 t2 ON t1.id = t2.id_foraneo

WHERE  t2.condicion = 'valor';

```

  

#### 🗄️ Categoría Estricta

**DML** — Elimina filas (datos), nunca la estructura.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 6: DELETE — Eliminar con seguridad

-- ══════════════════════════════════════════════════════

  

-- Insertar datos de prueba adicionales para el demo

INSERT INTO productos (nombre, precio, stock, id_categoria) VALUES

    ('Producto Obsoleto A', 5.00,  0, 1),

    ('Producto Obsoleto B', 8.00,  0, 2),

    ('Producto Obsoleto C', 3.00,  0, 3);

  

-- PASO 1: Siempre verificar con SELECT antes de DELETE

SELECT id_producto, nombre, stock

FROM   productos

WHERE  stock = 0;               -- los 3 productos obsoletos

  

-- PASO 2: Eliminar dentro de una transacción (seguridad)

START TRANSACTION;              -- iniciar transacción

  

    DELETE FROM productos

    WHERE  stock = 0;           -- elimina solo los sin stock

  

    -- Verificar que se eliminaron los correctos

    SELECT COUNT(*) AS total_restantes FROM productos;  -- debe ser 7

  

COMMIT;                         -- confirmar si todo está bien

-- ROLLBACK;                    -- o deshacer si algo falló

  

-- DELETE con subconsulta: eliminar productos de categorías inactivas

DELETE FROM productos

WHERE  id_categoria IN (

    SELECT id_categoria         -- subconsulta: obtener IDs de categorías inactivas

    FROM   categorias

    WHERE  activa = FALSE

);

  

-- Verificar que no quedan productos huérfanos de categorías inactivas

SELECT p.nombre, c.nombre AS categoria, c.activa

FROM   productos p

LEFT JOIN categorias c ON p.id_categoria = c.id_categoria

WHERE  c.activa = FALSE;       -- debe retornar 0 filas ✅

```
