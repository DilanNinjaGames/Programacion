### 5.2 `GROUP BY` y `HAVING`

#### ⚙️ Mecánica de la Query  

```

WHERE vs HAVING — la diferencia es CUÁNDO se aplica el filtro:

  

SIN HAVING:

  FROM → JOIN → WHERE (filtra filas) → GROUP BY (agrupa) → SELECT

  

CON HAVING:

  FROM → JOIN → WHERE (filtra filas) → GROUP BY (agrupa) → HAVING (filtra grupos) → SELECT

  

WHERE:  filtra FILAS antes de agrupar  → no puede usar funciones de agregación

HAVING: filtra GRUPOS después de agrupar → DEBE usar funciones de agregación

  

Ejemplo visual:

  Filas:        [E,100][E,200][R,50][E,300][R,80][D,150]

  WHERE E:      [E,100][E,200][E,300]  ← se aplica fila por fila

  GROUP BY cat: [[E,100][E,200][E,300]] [[R...]] [[D...]]

  HAVING >150:  [[E,100+200+300=600]] ← se aplica al GRUPO completo

```

  

#### 🔤 Sintaxis Estándar

  

```sql

SELECT   columna_grupo, funcion_agregada(col) AS alias

FROM     tabla

WHERE    condicion_de_fila          -- filtra antes de agrupar

GROUP BY columna_grupo              -- agrupa por esta columna

HAVING   funcion_agregada(col) > valor  -- filtra después de agrupar

ORDER BY funcion_agregada(col) DESC;

```

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 12: GROUP BY + HAVING — Análisis por categoría

-- ══════════════════════════════════════════════════════

  

-- Estadísticas de productos POR CATEGORÍA

SELECT

    c.nombre            AS categoria,

    COUNT(p.id_producto) AS total_productos,

    ROUND(AVG(p.precio), 2) AS precio_promedio,

    SUM(p.stock)         AS stock_total,

    MIN(p.precio)        AS precio_min,

    MAX(p.precio)        AS precio_max

FROM categorias c

LEFT JOIN productos p ON c.id_categoria = p.id_categoria

GROUP BY c.id_categoria, c.nombre    -- agrupa por categoría

ORDER BY stock_total DESC;

  

-- HAVING: solo categorías con MÁS de 1 producto

SELECT

    c.nombre            AS categoria,

    COUNT(p.id_producto) AS total_productos

FROM categorias c

INNER JOIN productos p ON c.id_categoria = p.id_categoria

GROUP BY c.id_categoria, c.nombre

HAVING COUNT(p.id_producto) > 1      -- filtra GRUPOS (no filas individuales)

ORDER BY total_productos DESC;

  

-- Diferencia WHERE vs HAVING:

-- WHERE: excluye filas antes de agrupar (precio > 30 elimina filas baratas)

-- HAVING: excluye grupos después de agrupar (promedio > 50 elimina categorías baratas)

  

SELECT

    c.nombre                    AS categoria,

    COUNT(p.id_producto)        AS productos_caros,   -- solo cuenta los > 30

    ROUND(AVG(p.precio), 2)     AS precio_promedio

FROM categorias c

INNER JOIN productos p ON c.id_categoria = p.id_categoria

WHERE  p.precio > 30                 -- ← filtra FILAS: excluye productos baratos

GROUP BY c.id_categoria, c.nombre

HAVING AVG(p.precio) > 50            -- ← filtra GRUPOS: solo categorías con promedio > 50

ORDER BY precio_promedio DESC;

  

-- Reporte de ventas por categoría (usando la tabla ventas creada antes)

SELECT

    c.nombre                        AS categoria,

    COUNT(v.id_venta)               AS total_ventas,

    SUM(v.cantidad)                 AS unidades_vendidas,

    SUM(v.cantidad * p.precio)      AS ingreso_total,

    ROUND(AVG(v.cantidad), 1)       AS promedio_por_venta

FROM ventas v

INNER JOIN productos   p ON v.id_producto  = p.id_producto

INNER JOIN categorias  c ON p.id_categoria = c.id_categoria

GROUP BY c.id_categoria, c.nombre

HAVING SUM(v.cantidad * p.precio) > 100   -- solo categorías con ingresos > $100

ORDER BY ingreso_total DESC;

```
