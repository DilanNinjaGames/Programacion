### 4.1 INNER JOIN

#### ⚙️ Mecánica de la Query

  

```

Tabla A (productos)    Tabla B (categorias)

──────────────────     ──────────────────────

id │ nombre │ id_cat   id_cat │ nombre

───┼─────────┼───────  ───────┼──────────────

1  │ Laptop  │ 1    ─────► 1  │ Electrónica  ✅ MATCH → aparece

2  │ Camisa  │ 2    ─────► 2  │ Ropa         ✅ MATCH → aparece

3  │ X-Prod  │ 99   ─────► ?? │ no existe    ❌ SIN MATCH → NO aparece

  

INNER JOIN: SOLO las filas donde existe coincidencia en AMBAS tablas

            Es la INTERSECCIÓN del diagrama de Venn

```

  

El motor recorre cada fila de la tabla izquierda, busca una coincidencia en la tabla derecha usando la condición del `ON`, y solo incluye el par cuando hay match.

  

#### 🔤 Sintaxis Estándar

  

```sql

SELECT t1.col, t2.col

FROM   tabla1 t1

INNER JOIN tabla2 t2 ON t1.clave_foranea = t2.clave_primaria;

  

-- Con USING (cuando las columnas tienen el mismo nombre en ambas tablas)

SELECT col1, col2

FROM   tabla1

INNER JOIN tabla2 USING (nombre_columna_comun);

```

  

#### 🗄️ Categoría Estricta

**DQL** — Es una cláusula de consulta, no modifica datos.

  

#### 💻 Laboratorio Práctico

  

```sql

-- ══════════════════════════════════════════════════════

-- LAB 8: INNER JOIN — Solo filas con coincidencia

-- ══════════════════════════════════════════════════════

  

-- JOIN básico: productos con su categoría

SELECT

    p.id_producto,

    p.nombre        AS producto,

    p.precio,

    p.stock,

    c.nombre        AS categoria        -- columna de la tabla derecha

FROM productos  p                       -- tabla izquierda (alias p)

INNER JOIN categorias c                 -- tabla derecha (alias c)

    ON p.id_categoria = c.id_categoria; -- condición de unión: FK = PK

-- Resultado: SOLO los productos que tienen una categoría existente

-- Los productos con id_categoria = NULL o inválido NO aparecen

  

-- INNER JOIN con filtros adicionales en WHERE

SELECT

    p.nombre        AS producto,

    p.precio,

    c.nombre        AS categoria

FROM productos p

INNER JOIN categorias c ON p.id_categoria = c.id_categoria

WHERE  c.activa = TRUE                  -- solo categorías activas

  AND  p.precio > 50                    -- solo productos con precio > 50

ORDER BY c.nombre, p.precio DESC;       -- ordenar por categoría, luego precio desc

  

-- INNER JOIN de tres tablas (encadenado)

-- Supongamos que tenemos una tabla de ventas

CREATE TABLE ventas (

    id_venta    INT AUTO_INCREMENT PRIMARY KEY,

    id_producto INT NOT NULL,

    cantidad    INT NOT NULL,

    fecha_venta DATE DEFAULT (CURRENT_DATE),

    FOREIGN KEY (id_producto) REFERENCES productos(id_producto)

);

  

INSERT INTO ventas (id_producto, cantidad) VALUES

    (1, 2), (2, 5), (1, 1), (3, 10), (6, 3);

  

-- JOIN triple: ventas → productos → categorías

SELECT

    v.id_venta,

    v.fecha_venta,

    v.cantidad,

    p.nombre        AS producto,

    p.precio,

    (v.cantidad * p.precio) AS total_venta,   -- columna calculada

    c.nombre        AS categoria

FROM ventas v

INNER JOIN productos   p ON v.id_producto   = p.id_producto    -- venta → producto

INNER JOIN categorias  c ON p.id_categoria  = c.id_categoria   -- producto → categoría

ORDER BY v.fecha_venta DESC;

```
