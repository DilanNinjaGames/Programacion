### 1.1 Tipado Dinámico, Indentación y Comentarios

Python usa **tipado dinámico**: una variable no tiene tipo fijo, el tipo lo tiene el *objeto* al que apunta. La **indentación** (4 espacios por convención PEP 8) es sintaxis obligatoria, no decorativa.

```python


# Comentario de una línea 

"""
Docstring / comentario multilínea.

Usado para documentar módulos, clases y funciones.

"""  

# Tipado dinámico: la variable 'x' puede apuntar a objetos de distinto tipo

x = 10          # x apunta a un int

x = "hola"      # ahora apunta a un str — completamente válido

x = [1, 2, 3]   # ahora apunta a una list

# La indentación define bloques — NO se usan llaves {}

if True:

    print("Bloque correcto")   # 4 espacios

    if True:

        print("Bloque anidado")  # 8 espacios

```

### 1.2 Variables, Tipos Nativos e Introspección


```python

# ── Tipos nativos más comunes ──────────────────────────────────────────────

entero    = 42

flotante  = 3.14

complejo  = 2 + 3j

texto     = "Python"

booleano  = True          # True / False (con mayúscula)

nulo      = None          # El equivalente a null

  

# ── Introspección en tiempo de ejecución ──────────────────────────────────

print(type(entero))           # <class 'int'>

print(type(texto))            # <class 'str'>

  

# isinstance() es preferible a type() para chequeos:

# — respeta herencia (un bool ES un int en Python)

print(isinstance(booleano, int))    # True  ← bool hereda de int

print(isinstance(booleano, bool))   # True

print(type(booleano) == int)        # False ← type() es exacto, no hereda

  

# ── Conversión de tipos (casting) ─────────────────────────────────────────

print(int("42"))        # 42

print(float("3.14"))    # 3.14

print(str(100))         # "100"

print(bool(0))          # False  — 0, "", [], {}, None son "falsy"

print(bool("hola"))     # True   — todo lo demás es "truthy"

```

### 1.3 Operadores Básicos

  

```python

# ── Aritméticos ────────────────────────────────────────────────────────────

print(17 // 5)    # 3    → división entera (floor division)

print(17 %  5)    # 2    → módulo (resto)

print(2  ** 8)    # 256  → potencia

print(17 /  5)    # 3.4  → división real (siempre float)

  

# ── Comparación ────────────────────────────────────────────────────────────

# Devuelven True o False

print(3 != 4)     # True

print(3 == 3.0)   # True  ← compara valor, no tipo

  

# ── Identidad vs Igualdad ─────────────────────────────────────────────────

a = [1, 2]

b = [1, 2]

c = a

  

print(a == b)     # True  — mismo contenido

print(a is b)     # False — objetos distintos en memoria

print(a is c)     # True  — c apunta al MISMO objeto que a

  

# ── Lógicos ────────────────────────────────────────────────────────────────

# and / or usan short-circuit: devuelven el valor que decidió el resultado

print(0 or "default")     # "default"  ← muy usado para valores por defecto

print("hola" and "mundo") # "mundo"    ← ambos truthy, devuelve el último

print(not True)           # False

```

### 1.4 Estructuras de Control y Operador Ternario

  

```python

# ── if / elif / else ───────────────────────────────────────────────────────

nota = 75

if nota >= 90:

    grado = "A"

elif nota >= 70:

    grado = "B"

else:

    grado = "C"

  

# ── Operador Ternario (inline if-else) ────────────────────────────────────

# Sintaxis: valor_si_true if condicion else valor_si_false

grado = "aprobado" if nota >= 60 else "reprobado"

print(grado)   # "aprobado"

  

# Anidado (usar con moderación — sacrifica legibilidad)

categoria = "alta" if nota >= 90 else ("media" if nota >= 70 else "baja")

  

# ── for / while ───────────────────────────────────────────────────────────

for i in range(5):          # range(inicio, fin_exclusivo, paso)

    if i == 3:

        continue            # salta esta iteración

    if i == 4:

        break               # sale del bucle

    print(i)                # 0, 1, 2

  

# ── Cláusula else en bucles ───────────────────────────────────────────────

# Se ejecuta SOLO si el bucle terminó sin break

for i in range(5):

    if i == 10:

        break

else:

    print("Bucle completado sin break")   # ← se imprime

  

# ── match / case (Python 3.10+) ───────────────────────────────────────────

comando = "salir"

match comando:

    case "iniciar":

        print("Iniciando...")

    case "salir" | "exit":       # OR con |

        print("Cerrando...")

    case _:                      # default

        print("Comando desconocido")

```

### 1.5 El Operador Walrus (`:=`)

  

El operador walrus (``:=``, Python 3.8+) **asigna y evalúa** en una sola expresión. Su nombre viene de los colmillos de una morsa. Elimina la necesidad de pre-computar un valor antes del condicional.

  

```python

# ── Problema sin walrus ────────────────────────────────────────────────────

import re

data = "Usuario: admin123"

  

resultado = re.search(r'\w+$', data)     # primero busco

if resultado:                             # luego evalúo

    print(resultado.group())             # y uso el resultado

  

# ── Solución con walrus (DRY) ─────────────────────────────────────────────

if m := re.search(r'\w+$', data):       # asigno Y evalúo en una línea

    print(m.group())   # "admin123"

  

# ── Caso de uso canónico: lectura de archivos en chunks ───────────────────

# Sin walrus:

chunk = f.read(8192)

while chunk:

    procesar(chunk)

    chunk = f.read(8192)     # ← duplicación

  

# Con walrus:

while chunk := f.read(8192):   # lee, asigna a chunk, y evalúa si es truthy

    procesar(chunk)

  

# ── En list comprehensions ─────────────────────────────────────────────────

# Útil cuando necesitas reutilizar un valor computado costoso dentro de la comp.

resultados = [y for x in range(10) if (y := x**2) > 20]

print(resultados)   # [25, 36, 49, 64, 81]

```
