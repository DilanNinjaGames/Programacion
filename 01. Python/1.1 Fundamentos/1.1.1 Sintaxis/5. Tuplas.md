### 5.1 Inmutabilidad, Eficiencia y Casos de Uso Reales

  

```python

# ── Creación ───────────────────────────────────────────────────────────────

t1 = (1, 2, 3)

t2 = 1, 2, 3       # los paréntesis son OPCIONALES — también es tupla

t3 = (42,)          # ← la coma es OBLIGATORIA para tupla de 1 elemento

t4 = (42)           # esto es solo un int entre paréntesis, NO tupla

  

# ── Inmutabilidad ─────────────────────────────────────────────────────────

t1[0] = 99   # ❌ TypeError: 'tuple' object does not support item assignment

  

# PERO si un elemento es mutable, ESE elemento puede cambiar internamente:

t5 = ([1, 2], [3, 4])

t5[0].append(99)

print(t5)   # ([1, 2, 99], [3, 4]) ← la lista interna cambió

  

# ── Ventajas de la inmutabilidad ──────────────────────────────────────────

# 1. RENDIMIENTO: las tuplas son más rápidas de crear e iterar que las listas

# 2. SEGURIDAD: garantiza que los datos no serán modificados accidentalmente

# 3. HASHABLE: pueden usarse como claves de diccionario o elementos de set

d = {(0, 0): "origen", (1, 0): "derecha"}   # ✅ válido

# d = {[0, 0]: "origen"}   # ❌ TypeError: las listas no son hashable

  

# ── Casos de uso reales ───────────────────────────────────────────────────

# Coordenadas (valores que van juntos y no deben cambiar)

punto  = (40.7128, -74.0060)    # lat, lon de Nueva York

  

# Retorno múltiple de funciones (Python retorna tuplas implícitas)

def min_max(lst):

    return min(lst), max(lst)   # retorna tupla sin paréntesis

  

minimo, maximo = min_max([3, 1, 4, 1, 5])

print(minimo, maximo)   # 1 5

  

# Registros heterogéneos (aunque namedtuple es mejor para esto)

persona = ("Ana", 30, "developer")

```

### 5.2 Unpacking Avanzado y Operador Asterisco

  

```python

# ── Unpacking básico ──────────────────────────────────────────────────────

x, y, z = (1, 2, 3)

a, b    = b, a           # swap sin variable temporal — muy pythónico

  

# ── Extended unpacking con * (Python 3) ──────────────────────────────────

primero, *resto   = [1, 2, 3, 4, 5]

print(primero)    # 1

print(resto)      # [2, 3, 4, 5]   ← * captura el sobrante como lista

  

primero, *medio, ultimo = [1, 2, 3, 4, 5]

print(primero)    # 1

print(medio)      # [2, 3, 4]

print(ultimo)     # 5

  

# Ignorar valores con _

_, segundo, *_ = (10, 20, 30, 40)

print(segundo)    # 20

  

# ── Unpacking en bucles ───────────────────────────────────────────────────

puntos = [(1, 2), (3, 4), (5, 6)]

for x, y in puntos:

    print(f"x={x}, y={y}")

  

# ── *args en funciones: recibe múltiples positionales como tupla ──────────

def suma(*args):

    """args es una TUPLA dentro de la función"""

    print(type(args))    # <class 'tuple'>

    return sum(args)

  

print(suma(1, 2, 3, 4))   # 10

  

# Desempacar al llamar: * expande un iterable como argumentos posicionales

numeros = [3, 1, 4]

print(max(*numeros))   # equivale a max(3, 1, 4)

```

### 5.3 Named Tuples

  

`namedtuple` crea una clase de tupla cuyos elementos tienen **nombre**. Combina la eficiencia de la tupla con la legibilidad de los atributos con nombre.

  

```python

from collections import namedtuple

  

# ── Definición ────────────────────────────────────────────────────────────

# namedtuple(NombreClase, [lista de campos])

Punto  = namedtuple('Punto', ['x', 'y'])

Punto3D = namedtuple('Punto3D', 'x y z')   # también acepta string separado por espacios

  

# ── Uso ───────────────────────────────────────────────────────────────────

p = Punto(x=3, y=4)

print(p.x)        # 3   — acceso por nombre (legible)

print(p[0])       # 3   — también acceso por índice (compatible con tupla)

print(p)          # Punto(x=3, y=4)

  

# ── Inmutable como cualquier tupla ────────────────────────────────────────

p.x = 10   # ❌ AttributeError

  

# Pero puedes crear una copia con valores modificados:

p2 = p._replace(x=10)

print(p2)   # Punto(x=10, y=4)

  

# ── Caso real: registro de empleados ──────────────────────────────────────

Empleado = namedtuple('Empleado', ['nombre', 'departamento', 'salario'])

  

empleados = [

    Empleado('Ana',  'IT',      70000),

    Empleado('Luis', 'Ventas',  45000),

    Empleado('Mia',  'IT',      80000),

]

  

# Código auto-documentado y legible

it_team = [e for e in empleados if e.departamento == 'IT']

salario_promedio = sum(e.salario for e in it_team) / len(it_team)

print(f"Promedio IT: {salario_promedio:,.0f}")   # 75,000

  

# ── _asdict() convierte a OrderedDict ─────────────────────────────────────

print(empleados[0]._asdict())

# {'nombre': 'Ana', 'departamento': 'IT', 'salario': 70000}

```
