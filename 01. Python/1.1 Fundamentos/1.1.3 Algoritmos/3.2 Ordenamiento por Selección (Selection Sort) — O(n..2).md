**Teoría:** Selection Sort es conceptualmente diferente a Bubble Sort. En lugar de intercambiar pares adyacentes repetidamente, en cada pasada **selecciona el elemento mínimo** del rango no ordenado y lo mueve directamente a su posición final.

  

**Visualización** sobre `[64, 25, 12, 22, 11]`:

  

```

Pasada 1: rango [64,25,12,22,11] → mínimo=11 (índice 4) → swap con índice 0

          → [11, 25, 12, 22, 64]  ← 11 en posición final ✅

  

Pasada 2: rango [25,12,22,64]    → mínimo=12 (índice 2) → swap con índice 1

          → [11, 12, 25, 22, 64]  ← 12 en posición final ✅

  

Pasada 3: rango [25,22,64]       → mínimo=22 (índice 3) → swap con índice 2

          → [11, 12, 22, 25, 64]  ← 22 en posición final ✅

  

Pasada 4: rango [25,64]          → mínimo=25 (índice 3) → swap con índice 3

          → [11, 12, 22, 25, 64]  ← ya estaba ✅

```

  

**Ventaja sobre Bubble Sort:** Hace exactamente `n-1` intercambios (uno por pasada), mientras que Bubble Sort puede hacer hasta $O(n^2)$ intercambios. Esto lo hace útil cuando la escritura en memoria es costosa (ej. memoria flash).

  

**Desventaja:** No tiene la optimización de "ya está ordenado" de Bubble Sort. Siempre hace $O(n^2)$ comparaciones, incluso si la lista ya está ordenada.

  

```python

# ══════════════════════════════════════════════════════════════════════════

# SELECTION SORT

# ══════════════════════════════════════════════════════════════════════════

  

def selection_sort(lista: list) -> list:

    """

    Ordena 'lista' encontrando el mínimo y poniéndolo en su posición correcta.

  

    Complejidad Temporal: O(n²) en TODOS los casos (sin optimización posible).

    Complejidad Espacial: O(1) — in-place, solo variables de control.

    Intercambios: exactamente n-1 (mínimo garantizado entre los O(n²) sorts).

    """

    n = len(lista)

  

    # El bucle externo representa la "frontera" entre ordenado y no ordenado

    # Después de la pasada i, los primeros i+1 elementos están ordenados

    for i in range(n - 1):

  

        # Asumimos que el primer elemento del rango no-ordenado es el mínimo

        indice_minimo = i

  

        # Buscamos si hay algún elemento más pequeño en el rango [i+1, n)

        for j in range(i + 1, n):

            if lista[j] < lista[indice_minimo]:

                indice_minimo = j   # actualizamos el índice del mínimo encontrado

  

        # Si encontramos un mínimo diferente al supuesto, intercambiamos

        # (si indice_minimo == i, el elemento ya está en su lugar → no swapeamos)

        if indice_minimo != i:

            lista[i], lista[indice_minimo] = lista[indice_minimo], lista[i]

  

    return lista

  
  

# ── Prueba y comparación con Bubble Sort ─────────────────────────────────

datos = [64, 25, 12, 22, 11]

print(f"Original:  {datos}")

print(f"Ordenada:  {selection_sort(datos.copy())}")

  
  

# ── Versión con traza visual ──────────────────────────────────────────────

def selection_sort_con_traza(lista: list) -> list:

    lista = lista.copy()

    n = len(lista)

    total_swaps = 0

  

    for i in range(n - 1):

        indice_minimo = i

        for j in range(i + 1, n):

            if lista[j] < lista[indice_minimo]:

                indice_minimo = j

  

        if indice_minimo != i:

            print(f"Pasada {i+1}: mínimo={lista[indice_minimo]} "

                  f"(índice {indice_minimo}) → swap con índice {i}")

            lista[i], lista[indice_minimo] = lista[indice_minimo], lista[i]

            total_swaps += 1

            print(f"         {lista}  ← posición {i} fijada ✅")

        else:

            print(f"Pasada {i+1}: {lista[i]} ya está en su lugar ✅")

  

    print(f"\nTotal de swaps: {total_swaps} (siempre ≤ n-1 = {n-1})")

    return lista

  

selection_sort_con_traza([64, 25, 12, 22, 11])

```
