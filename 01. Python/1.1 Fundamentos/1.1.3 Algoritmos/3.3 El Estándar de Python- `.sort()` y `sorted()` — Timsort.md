**Teoría — ¿Qué es Timsort?**

  

Python usa **Timsort**, un algoritmo híbrido creado por Tim Peters en 2002 específicamente para Python, hoy también usado en Java, Android y Swift. No es un algoritmo teórico puro: es una solución de ingeniería que explota patrones reales del mundo real.

  

Timsort combina dos algoritmos:

- **Insertion Sort** para fragmentos pequeños (hasta 64 elementos): simple y con muy bajo overhead en listas pequeñas.

- **Merge Sort** para combinar fragmentos grandes: divide-y-vencerás con complejidad garantizada $O(n \log n)$.

  

**La clave inteligente — Runs:** Timsort detecta **"runs"**: secuencias ya ordenadas (o inversamente ordenadas) que existen naturalmente en los datos del mundo real. Datos de fechas, nombres, IDs — casi siempre vienen parcialmente ordenados. Al aprovechar esto, Timsort logra $O(n)$ en listas ya ordenadas y se acerca a eso en datos parcialmente ordenados.

  

**Por qué siempre usar `.sort()` en lugar de escribir el tuyo:**

1. Está **escrito en C** — decenas de veces más rápido que cualquier Python puro.

2. Garantía $O(n \log n)$ en todos los casos.

3. Es **estable**: elementos iguales mantienen su orden relativo original.

4. Décadas de optimización y pruebas.

  

```python

# ══════════════════════════════════════════════════════════════════════════

# TIMSORT: .sort() y sorted()

# ══════════════════════════════════════════════════════════════════════════

  

# ── Diferencia fundamental ────────────────────────────────────────────────

numeros = [3, 1, 4, 1, 5, 9, 2, 6]

  

# .sort(): modifica IN-PLACE, retorna None

numeros.sort()

print(numeros)              # [1, 1, 2, 3, 4, 5, 6, 9] — original modificado

  

# sorted(): retorna una NUEVA lista, original intacta

original = [3, 1, 4, 1, 5, 9, 2, 6]

nueva    = sorted(original)

print(original)             # [3, 1, 4, 1, 5, 9, 2, 6] — sin cambios

print(nueva)                # [1, 1, 2, 3, 4, 5, 6, 9]

  

# ── Parámetros reverse y key ──────────────────────────────────────────────

# reverse=True: orden descendente

print(sorted([3,1,4,1,5], reverse=True))   # [5, 4, 3, 1, 1]

  

# key: función que extrae el criterio de comparación

# La función se llama UNA vez por elemento (eficiencia garantizada)

palabras = ["banana", "Kiwi", "manzana", "pera", "UVA"]

  

# Por longitud

print(sorted(palabras, key=len))

# ['Kiwi', 'UVA', 'pera', 'banana', 'manzana']

  

# Ignorando mayúsculas/minúsculas

print(sorted(palabras, key=str.lower))

# ['banana', 'Kiwi', 'manzana', 'pera', 'UVA']

  

# ── Ordenamiento estable: los iguales mantienen su orden ─────────────────

estudiantes = [

    {"nombre": "Ana",    "nota": 8},

    {"nombre": "Luis",   "nota": 9},

    {"nombre": "Beatriz","nota": 8},

    {"nombre": "Carlos", "nota": 7},

    {"nombre": "Diana",  "nota": 9},

]

  

# Ordenar por nota — los empates mantienen el orden original (estable)

por_nota = sorted(estudiantes, key=lambda e: e["nota"], reverse=True)

for e in por_nota:

    print(f"  {e['nombre']:10} → {e['nota']}")

# Luis       → 9   (Luis antes que Diana: orden original preservado)

# Diana      → 9

# Ana        → 8   (Ana antes que Beatriz: orden original preservado)

# Beatriz    → 8

# Carlos     → 7

  

# ── Ordenamiento por múltiples criterios ─────────────────────────────────

# Usando tupla como key: compara primer elemento, luego segundo si empatan

datos = [

    ("IT",     "Ana",  70000),

    ("Ventas", "Luis", 45000),

    ("IT",     "Mia",  80000),

    ("Ventas", "Pedro",55000),

]

# Primero por departamento, luego por salario descendente dentro de cada dept.

ordenado = sorted(datos, key=lambda x: (x[0], -x[2]))

for fila in ordenado:

    print(fila)

# ('IT', 'Mia', 80000)

# ('IT', 'Ana', 70000)

# ('Ventas', 'Pedro', 55000)

# ('Ventas', 'Luis', 45000)

  
  

# ── Comparación de velocidad: Bubble Sort vs Timsort ─────────────────────

import random, time

  

datos_prueba = random.sample(range(100_000), 10_000)

  

# Bubble Sort (O(n²) en C)

copia_bubble = datos_prueba.copy()

inicio = time.perf_counter()

bubble_sort(copia_bubble)

t_bubble = (time.perf_counter() - inicio) * 1000

  

# Timsort (O(n log n) en C)

copia_tim = datos_prueba.copy()

inicio = time.perf_counter()

copia_tim.sort()

t_tim = (time.perf_counter() - inicio) * 1000

  

print(f"Bubble Sort: {t_bubble:.1f}ms")   # ~800ms

print(f"Timsort:     {t_tim:.1f}ms")      # ~1ms

print(f"Timsort es {t_bubble/t_tim:.0f}x más rápido")   # ~800x

```
