### 4.1 Módulo `time`

  

**Teoría — Tiempo de CPU vs Tiempo Real (Wall Clock):**

  

Esta distinción es fundamental para benchmarking correcto:

  

- **Wall clock time (`time.time()`)**: Segundos transcurridos desde la Época Unix (1 enero 1970 UTC). Incluye tiempo de espera de I/O, tiempo cedido a otros procesos, y cambios de reloj del sistema. **Mide el tiempo que experimentas como usuario.**

- **CPU time (`time.process_time()`)**: Solo el tiempo que la CPU dedicó a **tu proceso**. Excluye sleeps y esperas. Útil para medir complejidad algorítmica pura.

- **Performance counter (`time.perf_counter()`)**: Reloj de **más alta resolución disponible** en el sistema. Incluye tiempo de sleep. Referencia no especificada (el cero es arbitrario). **El estándar para benchmarking de código.**

  

```python

import time

  

# ── Tres tipos de reloj ───────────────────────────────────────────────────

t_pared = time.time()           # float: segundos epoch (baja resolución)

t_cpu   = time.process_time()   # float: tiempo CPU del proceso (sin sleep)

t_perf  = time.perf_counter()   # float: alta resolución (nanosegundos en algunos OS)

  

# ── Diferencia práctica: sleep afecta wall clock, no CPU time ─────────────

inicio_pared = time.time()

inicio_cpu   = time.process_time()

inicio_perf  = time.perf_counter()

  

time.sleep(0.1)   # cedemos la CPU por 100ms

  

fin_pared = time.time()

fin_cpu   = time.process_time()

fin_perf  = time.perf_counter()

  

print(f"Wall clock:  {fin_pared - inicio_pared:.4f}s")  # ~0.1000s

print(f"CPU time:    {fin_cpu   - inicio_cpu  :.4f}s")  # ~0.0001s ← no incluyó el sleep

print(f"Perf counter:{fin_perf  - inicio_perf :.4f}s")  # ~0.1000s

  

# ── perf_counter: benchmark de algoritmos ────────────────────────────────

def medir(func, *args, repeticiones=5):

    """Mide el tiempo promedio de ejecución de una función"""

    tiempos = []

    for _ in range(repeticiones):

        inicio = time.perf_counter()

        func(*args)

        fin = time.perf_counter()

        tiempos.append(fin - inicio)

    promedio = sum(tiempos) / len(tiempos)

    minimo   = min(tiempos)

    print(f"Promedio: {promedio*1000:.3f}ms | Mínimo: {minimo*1000:.3f}ms")

    return minimo

  

# Comparar: suma con loop vs sum() built-in

def suma_loop(n):

    total = 0

    for i in range(n): total += i

    return total

  

def suma_builtin(n):

    return sum(range(n))

  

medir(suma_loop,    100_000)   # ~5ms

medir(suma_builtin, 100_000)   # ~1.5ms ← sum() es C bajo el capó

  

# ── Decorador de benchmarking ─────────────────────────────────────────────

import functools

  

def cronometrar(func):

    @functools.wraps(func)

    def wrapper(*args, **kwargs):

        inicio = time.perf_counter()

        resultado = func(*args, **kwargs)

        fin = time.perf_counter()

        print(f"[{func.__name__}] Ejecutado en {(fin-inicio)*1000:.3f}ms")

        return resultado

    return wrapper

  

@cronometrar

def ordenar_millon():

    import random

    datos = random.sample(range(1_000_000), 1_000_000)

    datos.sort()

  

ordenar_millon()   # [ordenar_millon] Ejecutado en ~180ms

  

# ── time.sleep: pausa del proceso ────────────────────────────────────────

time.sleep(1)      # pausa 1 segundo

time.sleep(0.05)   # pausa 50 milisegundos

  

# ── time.monotonic: reloj que nunca retrocede ─────────────────────────────

# A diferencia de time.time(), monotonic no se ve afectado por ajustes del

# reloj del sistema (NTP, horario de verano). Ideal para timeouts.

inicio = time.monotonic()

# ... operación ...

if time.monotonic() - inicio > 5:

    raise TimeoutError("Operación excedió 5 segundos")

```

### 4.2 Módulo `datetime`

  

**Teoría:** `datetime` proporciona clases para manipular fechas y horas. La distinción más importante del módulo es entre objetos **naive** (sin zona horaria) y **aware** (con zona horaria `tzinfo`). Mezclarlos causa errores sutiles en producción.

  

- **`date`**: Solo fecha (año, mes, día).

- **`time`**: Solo hora (hora, minuto, segundo, microsegundo).

- **`datetime`**: Combinación de fecha y hora.

- **`timedelta`**: Duración — permite aritmética entre datetimes.

  

```python

from datetime import datetime, date, time, timedelta, timezone

  

# ── Creación de objetos ───────────────────────────────────────────────────

hoy      = date.today()                           # fecha actual del sistema

ahora    = datetime.now()                          # naive: sin zona horaria

ahora_utc = datetime.now(tz=timezone.utc)         # aware: con UTC explícito

  

fecha    = date(2024, 6, 15)

hora     = time(14, 30, 0)

momento  = datetime(2024, 6, 15, 14, 30, 0)

  

print(ahora)         # 2025-01-15 10:23:45.123456

print(ahora.date())  # 2025-01-15

print(ahora.time())  # 10:23:45.123456

  

# ── Propiedades ───────────────────────────────────────────────────────────

d = datetime(2024, 3, 15, 9, 30, 45)

print(d.year, d.month, d.day)          # 2024 3 15

print(d.hour, d.minute, d.second)      # 9 30 45

print(d.weekday())                     # 4 (0=Lun, 6=Dom)

print(d.isoweekday())                  # 5 (1=Lun, 7=Dom — estándar ISO)

print(d.strftime('%A'))                # 'Friday'

  

# ── Aritmética con timedelta ──────────────────────────────────────────────

# timedelta: days, seconds, microseconds (todo se normaliza internamente)

delta_1semana = timedelta(weeks=1)

delta_90dias  = timedelta(days=90)

delta_complejo = timedelta(days=1, hours=3, minutes=30)

  

manana      = ahora + timedelta(days=1)

hace_1_hora = ahora - timedelta(hours=1)

plazo       = datetime(2025, 12, 31) - datetime.now()

print(f"Faltan {plazo.days} días para fin de año")

  

# Diferencia entre dos fechas → timedelta

cumple = date(1990, 5, 20)

edad_delta = date.today() - cumple

print(f"Edad en días: {edad_delta.days}")

print(f"Edad en años: {edad_delta.days // 365}")

  

# ── strftime: datetime → string (Format TIME) ────────────────────────────

# Mnemónico: str-f-time = "string FROM time"

d = datetime(2024, 6, 15, 14, 30, 5)

print(d.strftime('%Y-%m-%d'))           # '2024-06-15'  (ISO 8601)

print(d.strftime('%d/%m/%Y'))           # '15/06/2024'  (Europa)

print(d.strftime('%m/%d/%Y'))           # '06/15/2024'  (EE.UU.)

print(d.strftime('%Y-%m-%d %H:%M:%S')) # '2024-06-15 14:30:05'

print(d.strftime('%A, %d de %B de %Y')) # 'Saturday, 15 de June de 2024'

print(d.strftime('%I:%M %p'))           # '02:30 PM'  (12h)

  

# ── strptime: string → datetime (Parse TIME) ─────────────────────────────

# Mnemónico: str-p-time = "string PARSE time"

s1 = "2024-06-15"

s2 = "15/06/2024 14:30"

s3 = "June 15, 2024"

  

d1 = datetime.strptime(s1, '%Y-%m-%d')

d2 = datetime.strptime(s2, '%d/%m/%Y %H:%M')

d3 = datetime.strptime(s3, '%B %d, %Y')

  

# ── ISO 8601: formato universal para APIs ─────────────────────────────────

d = datetime(2024, 6, 15, 14, 30, 0, tzinfo=timezone.utc)

print(d.isoformat())                    # '2024-06-15T14:30:00+00:00'

d_parsed = datetime.fromisoformat('2024-06-15T14:30:00+00:00')

  

# ── Zonas horarias ────────────────────────────────────────────────────────

# Python 3.9+: use zoneinfo (sin dependencias externas)

from zoneinfo import ZoneInfo

  

utc      = timezone.utc

madrid   = ZoneInfo('Europe/Madrid')

ny       = ZoneInfo('America/New_York')

tokio    = ZoneInfo('Asia/Tokyo')

  

ahora_utc = datetime.now(tz=utc)

en_madrid = ahora_utc.astimezone(madrid)

en_ny     = ahora_utc.astimezone(ny)

  

print(f"UTC:    {ahora_utc.strftime('%H:%M')}")

print(f"Madrid: {en_madrid.strftime('%H:%M')}")

print(f"NY:     {en_ny.strftime('%H:%M')}")

  

# ⚠️ Regla de oro: almacena siempre en UTC, convierte al mostrar

```
