### 1.1 Módulo `sys`

  

**Teoría:** `sys` expone los parámetros y funciones que interactúan directamente con el **intérprete de Python** en ejecución. A diferencia de `os`, que habla con el sistema operativo subyacente, `sys` habla con la máquina virtual de CPython: su estado interno, su pila de ejecución y sus canales de I/O estándar.

  

Tres conceptos clave bajo el capó:

  

- **`sys.argv`**: Cuando el SO lanza un proceso Python, le pasa los argumentos como una lista de strings al intérprete antes de ejecutar el bytecode. `sys.argv[0]` es siempre el nombre del script.

- **`sys.stdin` / `sys.stdout`**: Son objetos de archivo que envuelven los descriptores de archivo 0 y 1 del proceso. En programación competitiva, reemplazar `input()` por `sys.stdin.readline()` elimina el overhead del buffer de la consola, siendo hasta **5x más rápido** en lecturas masivas.

- **`sys.getsizeof()`**: Retorna el tamaño en bytes del objeto Python en memoria (su cabecera de objeto + datos). No es recursivo: para listas, mide el array de punteros, no los objetos apuntados.

  

```python

import sys

  

# ── sys.argv: argumentos desde terminal ───────────────────────────────────

# Ejecución: python script.py archivo.txt --verbose 42

print(sys.argv)        # ['script.py', 'archivo.txt', '--verbose', '42']

print(sys.argv[0])     # 'script.py'   — nombre del script

print(sys.argv[1])     # 'archivo.txt' — primer argumento

  

# Patrón idiomático para scripts con validación de argumentos

if len(sys.argv) < 2:

    print(f"Uso: python {sys.argv[0]} <archivo>", file=sys.stderr)

    sys.exit(1)   # código de salida 1 = error (0 = éxito)

  

nombre_archivo = sys.argv[1]

  

# ── sys.stdin.readline(): I/O ultrarrápido para competitiva ───────────────

# input() es lento porque limpia el buffer y procesa señales.

# sys.stdin.readline() lee directamente del buffer del SO.

import sys

input_rapido = sys.stdin.readline   # aliás al método

  

# Leer un entero

n = int(input_rapido())

# Leer lista de enteros en una línea: "1 2 3 4 5\n"

lista = list(map(int, input_rapido().split()))

  

# Para entrada masiva (miles de líneas), leer TODO de una vez es aún más rápido

data = sys.stdin.read().split()   # toda la entrada como lista de tokens

  

# ── sys.getsizeof(): introspección de memoria ──────────────────────────────

# ⚠️ NO es recursivo — mide solo el contenedor, no sus elementos

  

lista_vacia   = []

lista_10      = list(range(10))

lista_1000    = list(range(1000))

  

print(sys.getsizeof(lista_vacia))    # 56 bytes  (cabecera + array vacío)

print(sys.getsizeof(lista_10))       # 184 bytes (56 + 10 punteros × 8 bytes)

print(sys.getsizeof(lista_1000))     # 8056 bytes

  

# Comparación: lista vs generador

print(sys.getsizeof([x**2 for x in range(10_000)]))  # ~85,176 bytes

print(sys.getsizeof(x**2 for x in range(10_000)))    # 112 bytes ← lazy

  

# ── sys.path: rutas de búsqueda de módulos ───────────────────────────────

# Python busca imports en este orden: directorio actual → sys.path

print(sys.path)                    # lista de directorios

sys.path.insert(0, '/mi/modulo')   # agregar ruta personalizada al inicio

  

# ── Información del sistema ───────────────────────────────────────────────

print(sys.version)        # '3.12.0 (main, ...)'

print(sys.platform)       # 'linux', 'darwin', 'win32'

print(sys.maxsize)        # 9223372036854775807 (2^63 - 1 en 64-bit)

print(sys.float_info)     # precisión y límites de float

print(sys.getrecursionlimit())    # 1000 (límite por defecto)

sys.setrecursionlimit(10_000)     # aumentar para algoritmos profundos

```

### 1.2 Módulo `os`

  

**Teoría:** `os` es la capa de abstracción entre Python y las **llamadas al sistema (syscalls)** del SO. Su filosofía es la portabilidad: el mismo código `os.makedirs()` funciona en Linux, macOS y Windows, aunque internamente ejecute syscalls diferentes (`mkdir` en POSIX, `CreateDirectory` en Win32).

  

Concepto crítico — **`os.path` vs `pathlib`**: `os.path` trabaja con strings crudas y funciones procedurales. `pathlib` (sección 2) envuelve esas mismas operaciones en objetos `Path` orientados a objetos. Para código nuevo, usa `pathlib`; `os.path` lo encontrarás en código legacy.

  

```python

import os

  

# ── Directorio de trabajo ─────────────────────────────────────────────────

print(os.getcwd())            # '/home/usuario/proyecto'

os.chdir('/tmp')              # cambia el directorio de trabajo del proceso

  

# ── Gestión de directorios ────────────────────────────────────────────────

os.mkdir('nuevo_dir')                        # crea UN directorio (falla si ya existe)

os.makedirs('a/b/c', exist_ok=True)          # crea árbol completo (no falla si existe)

os.rmdir('nuevo_dir')                        # elimina directorio VACÍO

os.removedirs('a/b/c')                       # elimina árbol si queda vacío

  

# ── Gestión de archivos ───────────────────────────────────────────────────

os.rename('viejo.txt', 'nuevo.txt')          # renombrar/mover

os.remove('archivo.txt')                     # eliminar archivo

  

# ── Listar contenido ──────────────────────────────────────────────────────

print(os.listdir('.'))        # ['archivo.py', 'datos/', ...] — sin orden garantizado

  

# os.walk: traversal recursivo del árbol de directorios

# Genera tuplas (directorio_raiz, [subdirectorios], [archivos])

for raiz, dirs, archivos in os.walk('/mi/proyecto'):

    nivel = raiz.replace('/mi/proyecto', '').count(os.sep)

    indent = '  ' * nivel

    print(f"{indent}{os.path.basename(raiz)}/")

    for archivo in archivos:

        print(f"{indent}  {archivo}")

  

# ── os.path: manipulación de rutas (estilo legacy) ───────────────────────

ruta = '/home/usuario/proyecto/datos/archivo.csv'

  

print(os.path.dirname(ruta))    # '/home/usuario/proyecto/datos'

print(os.path.basename(ruta))   # 'archivo.csv'

print(os.path.splitext(ruta))   # ('/home/.../archivo', '.csv')

print(os.path.exists(ruta))     # True / False

print(os.path.isfile(ruta))     # True si es archivo

print(os.path.isdir(ruta))      # True si es directorio

print(os.path.getsize(ruta))    # tamaño en bytes

  

# Construcción de rutas portable (no usar "/" hardcodeado)

ruta_construida = os.path.join('/home', 'usuario', 'proyecto', 'main.py')

# En Windows usa \, en Unix usa /

  

# ── Variables de entorno ──────────────────────────────────────────────────

# os.environ es un objeto tipo diccionario que refleja el entorno del proceso

# Los cambios NO afectan al proceso padre (shell) — solo al proceso Python

  

print(os.environ.get('HOME'))             # '/home/usuario'

print(os.environ.get('API_KEY', 'no configurada'))  # valor por defecto

  

# Patrón estándar para configuración sensible (nunca hardcodear credenciales)

import os

DB_PASSWORD = os.environ.get('DB_PASSWORD')

if not DB_PASSWORD:

    raise EnvironmentError("Falta variable de entorno DB_PASSWORD")

  

os.environ['MI_VAR'] = 'valor'    # establecer variable para subprocesos

  

# ── Información del sistema ───────────────────────────────────────────────

print(os.sep)         # '/' en Unix, '\\' en Windows

print(os.linesep)     # '\n' en Unix, '\r\n' en Windows

print(os.cpu_count()) # número de CPUs lógicas (útil para threading/multiprocessing)

print(os.getpid())    # PID del proceso actual

```
