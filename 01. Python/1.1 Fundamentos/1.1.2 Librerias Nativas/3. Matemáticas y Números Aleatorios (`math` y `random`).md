### 3.1 Módulo `math`

  

**Teoría:** `math` expone las funciones matemáticas de la librería estándar de C (`libm`). Opera exclusivamente sobre números reales (`float`); para aritmética con **precisión arbitraria** usa `decimal.Decimal`, y para **números complejos** usa `cmath`.

  

Conceptos críticos:

- **`math.floor` / `math.ceil`** retornan `int` (desde Python 3), a diferencia de `int(x)` que trunca hacia cero (diferente para negativos: `int(-2.7)` → `-2`, `math.floor(-2.7)` → `-3`).

- **`math.inf`** es el valor flotante IEEE 754 para infinito. Útil como centinela en algoritmos de optimización (Dijkstra, Floyd-Warshall).

- **`math.log`**: Por defecto es logaritmo natural (base *e*). Acepta base como segundo argumento.

  

```python

import math

  

# ── Constantes ────────────────────────────────────────────────────────────

print(math.pi)      # 3.141592653589793

print(math.e)       # 2.718281828459045

print(math.tau)     # 6.283185307179586 (= 2π, útil en rotaciones)

print(math.inf)     # inf

print(math.nan)     # nan

print(math.isnan(math.nan))    # True

print(math.isinf(math.inf))    # True

  

# ── Redondeo — diferencia crítica con negativos ───────────────────────────

x = -2.7

print(int(x))          # -2  ← trunca hacia CERO

print(math.trunc(x))   # -2  ← equivalente a int()

print(math.floor(x))   # -3  ← hacia MENOS infinito (floor = suelo)

print(math.ceil(x))    # -2  ← hacia MÁS infinito  (ceil  = techo)

  

# Para positivos, floor/int coinciden; para negativos, divergen

print(math.floor(2.7))   # 2

print(math.ceil(2.3))    # 3

  

# ── Funciones de potencia y raíz ──────────────────────────────────────────

print(math.sqrt(144))      # 12.0  — raíz cuadrada (más rápida que ** 0.5)

print(math.isqrt(17))      # 4     — raíz cuadrada entera (≤ resultado real)

print(math.pow(2, 10))     # 1024.0 — siempre float (diferente de ** que es int)

print(math.exp(1))         # 2.718... — e^x

  

# ── Logaritmos ────────────────────────────────────────────────────────────

print(math.log(math.e))    # 1.0           — logaritmo natural (base e)

print(math.log(100, 10))   # 2.0           — log base 10

print(math.log2(1024))     # 10.0          — log base 2 (más preciso que log(x, 2))

print(math.log10(1000))    # 3.0           — log base 10 (más preciso que log(x, 10))

  

# Uso real: calcular bits necesarios para representar N valores

N = 1_000_000

bits_necesarios = math.ceil(math.log2(N))

print(f"Necesitas {bits_necesarios} bits para {N} valores")   # 20 bits

  

# ── Trigonometría (en RADIANES) ───────────────────────────────────────────

print(math.sin(math.pi / 2))    # 1.0

print(math.cos(0))              # 1.0

print(math.tan(math.pi / 4))    # 1.0 (≈)

  

# Conversión de unidades

grados  = 90

rad     = math.radians(grados)   # 1.5707... (π/2)

print(math.degrees(rad))          # 90.0

  

# ── Funciones de utilidad ────────────────────────────────────────────────

print(math.factorial(10))         # 3628800

print(math.gcd(48, 18))           # 6 — Máximo Común Divisor

print(math.lcm(4, 6))             # 12 — Mínimo Común Múltiplo (Python 3.9+)

print(math.comb(10, 3))           # 120 — combinaciones sin repetición C(n,k)

print(math.perm(10, 3))           # 720 — permutaciones P(n,k)

  

# math.fsum: suma con mayor precisión para flotantes (evita error acumulado)

datos = [0.1] * 10

print(sum(datos))        # 0.9999999999999999 ← error de punto flotante

print(math.fsum(datos))  # 1.0                ← compensado por Kahan summation

  

# ── Caso real: distancia euclidiana (robótica / algoritmos) ──────────────

def distancia(p1, p2):

    """Distancia entre dos puntos en N dimensiones"""

    return math.sqrt(sum((a - b)**2 for a, b in zip(p1, p2)))

  

# Python 3.8+: math.dist() hace exactamente esto, más eficiente

print(math.dist((0, 0), (3, 4)))   # 5.0

print(math.dist((1,2,3), (4,6,3))) # 5.0 (3D)

```

### 3.2 Módulo `random`

  

**Teoría:** `random` implementa el algoritmo **Mersenne Twister (MT19937)**, un generador de números pseudoaleatorios (PRNG) con un período de 2^19937−1. Es extremadamente rápido y pasa la mayoría de pruebas estadísticas, pero es **determinístico**: dado el mismo estado inicial (`seed`), produce la misma secuencia.

  

Consecuencia crítica: **nunca usar `random` para criptografía o seguridad**. Para eso, usar `secrets` (que usa `/dev/urandom` del SO, impredecible). `random` es para simulaciones, juegos, muestreo estadístico y tests reproducibles.

  

```python

import random

  

# ── Seed: reproducibilidad ────────────────────────────────────────────────

# Con la misma seed, la secuencia es idéntica — esencial para tests y ML

random.seed(42)

print(random.random())   # siempre 0.6394267984578837

random.seed(42)

print(random.random())   # mismo resultado: 0.6394267984578837

  

# Sin seed, usa el tiempo del sistema como fuente de entropía

  

# ── Números aleatorios ────────────────────────────────────────────────────

print(random.random())           # float en [0.0, 1.0)

print(random.uniform(1.5, 3.5)) # float en [a, b] (ambos inclusive)

print(random.randint(1, 10))     # int en [a, b] (ambos inclusive) ← ojo, diferente a range

print(random.randrange(0, 10, 2))# int de range(0,10,2): 0,2,4,6,8

  

# ── Distribuciones estadísticas ───────────────────────────────────────────

print(random.gauss(mu=0, sigma=1))       # distribución normal estándar

print(random.normalvariate(mu=0, sigma=1))  # igual pero thread-safe

print(random.expovariate(lambd=1/5))    # distribución exponencial (lambd=tasa)

  

# ── Selección de secuencias ───────────────────────────────────────────────

colores = ['rojo', 'verde', 'azul', 'amarillo', 'blanco']

  

# choice: un elemento (con reemplazo implícito)

print(random.choice(colores))      # 'verde' (aleatorio)

  

# choices: k elementos CON reemplazo (puede repetir) — soporte pesos

print(random.choices(colores, k=3))

# Con pesos: simular una moneda cargada (70% cara)

resultado = random.choices(['cara', 'sello'], weights=[70, 30], k=10)

print(resultado.count('cara'))     # aprox. 7

  

# sample: k elementos SIN reemplazo (no repite) — muestra estadística

print(random.sample(colores, k=3))  # 3 únicos

  

# ── shuffle: desordenar in-place (algoritmo Fisher-Yates) ────────────────

mazo = list(range(1, 53))

random.shuffle(mazo)     # modifica mazo directamente, O(n)

print(mazo[:5])          # [34, 7, 45, 2, 19] (aleatorio)

  

# Para no modificar el original, combinar con sample:

mazo_mezclado = random.sample(mazo, len(mazo))

  

# ── Caso real: simulación Monte Carlo para estimar π ─────────────────────

def estimar_pi(n_muestras: int) -> float:

    """

    Genera puntos aleatorios en el cuadrado [-1,1]×[-1,1].

    La fracción dentro del círculo unitario converge a π/4.

    """

    dentro = sum(

        1 for _ in range(n_muestras)

        if random.uniform(-1, 1)**2 + random.uniform(-1, 1)**2 <= 1

    )

    return 4 * dentro / n_muestras

  

random.seed(0)

print(f"π ≈ {estimar_pi(1_000_000):.5f}")   # ≈ 3.14159
