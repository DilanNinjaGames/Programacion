**Teoría:** `pathlib` (Python 3.4+) reemplaza el enfoque procedural de `os.path` con una **API orientada a objetos**. La clase central `Path` representa una ruta del sistema de archivos y expone operaciones como métodos y propiedades del objeto, en lugar de funciones sueltas.

  

Bajo el capó, `Path` es una clase abstracta. Al instanciarla, Python detecta el SO y retorna automáticamente una `PosixPath` (Linux/macOS) o `WindowsPath`. Esto garantiza portabilidad sin condicionales `if sys.platform`.

  

El operador `/` está sobrecargado (`__truediv__`) para concatenar segmentos de ruta, produciendo código que se lee exactamente como se ve una ruta en el sistema de archivos.

  

```python

from pathlib import Path

  

# ── Creación de rutas ──────────────────────────────────────────────────────

p_abs  = Path('/home/usuario/proyecto')     # ruta absoluta

p_rel  = Path('datos/archivo.csv')          # ruta relativa al CWD

p_home = Path.home()                        # directorio home del usuario

p_cwd  = Path.cwd()                         # directorio de trabajo actual

  

# ── El operador / para concatenar (más legible que os.path.join) ──────────

proyecto  = Path('/home/usuario/proyecto')

datos     = proyecto / 'datos'              # Path('/home/usuario/proyecto/datos')

archivo   = datos / 'ventas.csv'            # Path('.../datos/ventas.csv')

config    = proyecto / 'config' / 'app.json'

  

# ── Propiedades del objeto Path ───────────────────────────────────────────

p = Path('/home/usuario/proyecto/datos/reporte.final.csv')

  

print(p.name)       # 'reporte.final.csv'  — nombre con extensión

print(p.stem)       # 'reporte.final'       — sin la última extensión

print(p.suffix)     # '.csv'               — última extensión

print(p.suffixes)   # ['.final', '.csv']   — todas las extensiones

print(p.parent)     # Path('.../datos')    — directorio contenedor

print(p.parents[1]) # Path('.../proyecto') — abuelo

print(p.parts)      # ('/', 'home', 'usuario', 'proyecto', 'datos', 'reporte.final.csv')

  

# ── Verificación de existencia y tipo ────────────────────────────────────

print(p.exists())   # True / False

print(p.is_file())  # True si es archivo regular

print(p.is_dir())   # True si es directorio

print(p.stat().st_size)    # tamaño en bytes

print(p.stat().st_mtime)   # timestamp de última modificación

  

# ── Lectura y escritura directa (sin open() explícito) ───────────────────

# Ideal para archivos pequeños — una sola línea en lugar de context manager

archivo = Path('notas.txt')

archivo.write_text("Hola, mundo\n", encoding='utf-8')   # crea o sobreescribe

contenido = archivo.read_text(encoding='utf-8')          # lee todo como str

  

# Para binarios

archivo_bin = Path('imagen.png')

bytes_data  = archivo_bin.read_bytes()

  

# ── Creación de directorios ───────────────────────────────────────────────

nuevo = Path('proyecto/src/utils')

nuevo.mkdir(parents=True, exist_ok=True)   # equivale a os.makedirs(..., exist_ok=True)

  

# ── Renombrar y eliminar ──────────────────────────────────────────────────

p_old = Path('viejo.txt')

p_new = p_old.rename('nuevo.txt')         # retorna el Path nuevo

p_new.unlink()                             # elimina archivo

Path('directorio_vacio').rmdir()           # elimina dir vacío

  

# ── Navegación y glob ─────────────────────────────────────────────────────

directorio = Path('/mi/proyecto')

  

# Hijos directos (no recursivo)

for hijo in directorio.iterdir():

    print(hijo)

  

# glob: busca con patrones — retorna generador (lazy, eficiente)

for py_file in directorio.glob('*.py'):        # archivos .py en el directorio

    print(py_file)

  

for py_file in directorio.rglob('*.py'):       # recursivo (rglob = recursive glob)

    print(py_file)

  

# Caso real: recopilar todos los .csv en un árbol de directorios

csvs = sorted(Path('datos').rglob('*.csv'))

print(f"Encontrados {len(csvs)} archivos CSV")

  

# ── Resolución y conversión ───────────────────────────────────────────────

p_rel = Path('../../proyecto/main.py')

print(p_rel.resolve())    # ruta absoluta canónica (resuelve .. y symlinks)

print(str(p_rel))         # convertir a string cuando lo exige una API antigua

  

# ── Patrón: construir rutas relativas al script actual ───────────────────

# __file__ es la ruta del script en ejecución

BASE_DIR = Path(__file__).parent          # directorio del script

CONFIG   = BASE_DIR / 'config' / 'app.json'

LOGS     = BASE_DIR.parent / 'logs'      # un nivel arriba

```
