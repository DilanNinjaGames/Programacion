## 1. Configuración Básica y Sintaxis Fundamental

### 1.1. Estructura de un Script en Godot (`extends`)

Todo script de GDScript **debe declarar de qué nodo hereda** como primera línea. Esto es equivalente a que cada script fuera una clase que extiende un tipo de nodo de Godot.

```gdscript
# En Python no hay equivalente directo — en Godot es obligatorio
extends Node          # hereda de Node (nodo base)
extends Node2D        # para nodos 2D con posición/rotación
extends CharacterBody2D  # para personajes con física
extends Control       # para elementos de UI

# El script COMPLETO es implícitamente la clase del nodo
# No hay un bloque class: exterior — el archivo entero ES la clase
```

**Estructura típica de un script completo:**

```gdscript
extends CharacterBody2D   # ← primera línea siempre (equivale a class MiChar(CharacterBody2D):)

# --- Constantes y enums al inicio ---
const VELOCIDAD_MAX: float = 300.0

# --- Variables exportadas (visibles en el Inspector de Godot) ---
@export var vida: int = 100

# --- Variables privadas del nodo ---
var _direccion: Vector2 = Vector2.ZERO
@onready var _sprite: Sprite2D = $Sprite2D  # acceso a nodo hijo

# --- Señales personalizadas ---
signal vida_cambiada(nueva_vida: int)

# --- Ciclo de vida ---
func _ready() -> void:
    print("Nodo listo: ", name)

func _process(delta: float) -> void:
    pass
```

### 1.2. Variables (`var`), Constantes (`const`) y Tipado Estático vs Dinámico

GDScript soporta tipado dinámico (como Python) y tipado estático (como Python con type hints, pero **reforzado en tiempo de compilación**).

```gdscript
# ── TIPADO DINÁMICO (como Python puro) ────────────────────────────────────
var nombre = "Héroe"     # GDScript infiere String
var vida   = 100         # infiere int
var velocidad = 5.5      # infiere float

# ── TIPADO ESTÁTICO (recomendado en Godot 4 — mejor rendimiento) ──────────
var nombre: String  = "Héroe"
var vida: int       = 100
var velocidad: float = 5.5
var activo: bool    = true
var posicion: Vector2 = Vector2(0, 0)   # tipo nativo de Godot

# Inferencia con := (infiere el tipo y lo fija — como walrus pero solo para declarar)
var puntuacion := 0       # infiere int y lo fija como int estático
var texto := "hola"       # infiere String y lo fija

# ── CONSTANTES ────────────────────────────────────────────────────────────
const GRAVEDAD: float = 980.0
const NOMBRE_JUEGO  = "Mi Juego"   # también puede ser sin tipo
const TILE_SIZE: int = 16

# ── VARIABLES ESTÁTICAS (compartidas entre todas las instancias) ──────────
# Equivale a atributos de clase en Python
static var instancias_activas: int = 0

# ── TIPOS NATIVOS DE GODOT (no existen en Python estándar) ────────────────
var vec2: Vector2   = Vector2(100.0, 50.0)   # equivale a namedtuple(x, y)
var vec3: Vector3   = Vector3(1, 0, 0)
var color: Color    = Color(1.0, 0.5, 0.0, 1.0)  # RGBA en rango 0.0-1.0
var rect: Rect2     = Rect2(0, 0, 640, 480)
var nodo_ref: Node  = null                   # referencia tipada a nodo (puede ser null)
```

### 1.3. Diferencias Clave de Sintaxis con Python

```gdscript
# ── LO QUE ES IGUAL ───────────────────────────────────────────────────────
# Indentación por tabs (Godot usa tabs, Python usa 4 espacios por convención)
# Comentarios con #
# pass funciona igual
# and, or, not (no &&, ||, !)   ← GDScript usa las palabras, no los símbolos
# print(), len(), range()

# ── LO QUE CAMBIA ─────────────────────────────────────────────────────────

# 1. No hay None → es null
var ref = null              # Python: ref = None

# 2. No hay True/False con mayúscula → son true/false en minúscula
var activo: bool = true     # Python: activo = True
var pausado: bool = false   # Python: pausado = False

# 3. No hay listas con [] como tipo — se llaman Array
var lista: Array = [1, 2, 3]          # Python: lista = [1, 2, 3]
var tipado: Array[int] = [1, 2, 3]    # Python: lista: list[int] = [1, 2, 3]

# 4. No hay dict literal con tipo — se llama Dictionary
var config: Dictionary = {"vida": 100, "mana": 50}

# 5. No hay f-strings — se usa concatenación o str()
var msg = "Vida: " + str(vida)        # Python: f"Vida: {vida}"

# GDScript 4 sí tiene interpolación básica (sin el prefijo f):
# Godot 4.3+ soporta: var msg = "Vida: %d" % vida
var msg2 = "Vida: %d, Mana: %d" % [vida, mana]   # similar a % de Python

# 6. Los operadores de incremento ++ y -- NO existen
vida += 1    # correcto
# vida++     # ← ERROR en GDScript

# 7. is para verificar tipo (igual que Python)
if vida is int:
    print("es entero")

# 8. La negación es 'not', no '!'
if not activo:
    pass

# 9. No existe el operador ternario Python-style
# Python: x = a if condicion else b
# GDScript: NO EXISTE — usar if/else normal o hacky workaround
var resultado
if condicion:
    resultado = a
else:
    resultado = b
```
