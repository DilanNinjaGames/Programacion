# Java desde Python — Sección 1 y 2

## 🏗️ Fundamentos del Ecosistema Java (JDK, JRE, JVM)

### Diferencias prácticas entre JDK, JRE y JVM

#### 🔄 Contraste de Sintaxis

En Python, instalas un solo intérprete (`python3`) y listo: ese binario **compila y ejecuta** en el mismo paso, directamente sobre tu OS. Java separa ese proceso en tres capas con responsabilidades distintas:

```
┌─────────────────────────────────────────┐
│  JDK  (Java Development Kit)            │  ← Tú instalas esto para desarrollar
│  ┌───────────────────────────────────┐  │
│  │  JRE  (Java Runtime Environment) │  │  ← Solo para ejecutar (usuarios finales)
│  │  ┌─────────────────────────────┐ │  │
│  │  │  JVM  (Java Virtual Machine)│ │  │  ← El intérprete real del bytecode
│  │  └─────────────────────────────┘ │  │
│  └───────────────────────────────────┘  │
│  + javac (compilador) + herramientas    │
└─────────────────────────────────────────┘
```

|Componente|Qué es|Analogía Python|
|---|---|---|
|**JVM**|Máquina virtual que ejecuta bytecode `.class`|El intérprete `python3` en runtime|
|**JRE**|JVM + librerías estándar (rt.jar)|`python3` + stdlib instalada|
|**JDK**|JRE + `javac` + debugger + `javadoc` + profiler|`python3` + `pip` + `venv` + herramientas|

**La diferencia filosófica clave:** Python es _interpreted_ (el `.py` se lee línea a línea). Java es _compiled-then-interpreted_: `javac` produce bytecode primero, luego la JVM lo ejecuta. Esto da a Java verificación de tipos en **tiempo de compilación**, no en runtime.
