## 🔧 Herramientas de Calidad y Diagnóstico

### Debugging en IntelliJ IDEA

#### 🔄 Contraste de Sintaxis

En Python debuggeas con `print()`, `pdb.set_trace()` o el debugger de VS Code. En Java, IntelliJ IDEA tiene un debugger integrado de primera clase que es el estándar de la industria. La diferencia clave: **en Python el debugger se conecta al intérprete en vivo; en Java se conecta a la JVM**, que expone una interfaz de debug (JDWP — Java Debug Wire Protocol).

|Concepto|Python (pdb / VS Code)|Java (IntelliJ)|
|---|---|---|
|Punto de parada|`breakpoint()` en código|Click en margen izquierdo → punto rojo|
|Ejecutar línea a línea|`n` (next)|**F8** (Step Over)|
|Entrar a un método|`s` (step)|**F7** (Step Into)|
|Salir del método actual|`r` (return)|**Shift+F8** (Step Out)|
|Ver variables activas|`locals()` / panel Variables|Panel **Variables** automático|
|Evaluar expresión|`p expresion`|**Alt+F8** → Evaluate Expression|
|Continuar hasta el siguiente break|`c` (continue)|**F9** (Resume)|

**Breakpoints condicionales** — la ventaja real de un debugger IDE:

```
Click derecho sobre el punto rojo → "Edit Breakpoint"
→ Condition: i == 500
→ El programa se pausa SOLO cuando i vale 500
```

En Python tendrías que escribir `if i == 500: breakpoint()` manualmente en el código.

#### 💻 Bloque de Código — Código diseñado para practicar debugging

java

```java
// DepuracionDemo.java
// Abre este archivo en IntelliJ y coloca breakpoints en las líneas marcadas

public class DepuracionDemo {

    // Método con un bug intencional para practicar Step Into (F7)
    public static int calcularFactorial(int n) {
        if (n <= 0) return 1;                    // ← BREAKPOINT aquí (F8 para avanzar)
        int resultado = 1;
        for (int i = 1; i <= n; i++) {           // ← observa 'i' en panel Variables
            resultado *= i;                       // ← watch: resultado, i
        }
        return resultado;
    }

    public static void main(String[] args) {

        // Paso 1: coloca un breakpoint en la línea siguiente (click en el margen)
        int[] numeros = {1, 2, 3, 4, 5, 6, 7};  // ← BREAKPOINT SUGERIDO

        int suma = 0;
        for (int i = 0; i < numeros.length; i++) {
            suma += numeros[i];
            // En el panel Variables verás 'i', 'suma' y 'numeros' actualizarse en vivo
        }

        // Paso 2: F7 aquí entrará DENTRO de calcularFactorial
        int fact5 = calcularFactorial(5);         // ← usa Step Into (F7) aquí

        System.out.println("Suma: "       + suma);
        System.out.println("5! = "        + fact5);

        // Paso 3: breakpoint condicional — click derecho → condición: i == 3
        for (int i = 0; i < numeros.length; i++) {
            System.out.println("numeros[" + i + "] = " + numeros[i]);
        }
    }
}
```

#### 🪲 "Rompe el Código"

java

```java
// ❌ ERROR CLÁSICO: confiar en print() para debug de concurrencia o flujos complejos

public class BugSutil {
    public static void main(String[] args) {

        int[] datos = {10, 20, 30, 40, 50};
        int suma = 0;

        // El pythonista agrega prints y aun así no ve el bug:
        for (int i = 0; i <= datos.length; i++) {   // ← bug: <= en vez de <
            System.out.println("Procesando índice: " + i); // print no ayuda
            suma += datos[i];   // ArrayIndexOutOfBoundsException cuando i=5
        }
    }
}

// ❌ POR QUÉ OCURRE:
// datos.length = 5, pero los índices válidos son 0..4.
// La condición i <= datos.length permite i=5, que no existe.
// Un print antes del acceso te muestra "Procesando índice: 5"
// pero para entonces el crash ya es inevitable.
//
// ✅ CÓMO EL DEBUGGER LO RESUELVE:
// Breakpoint en 'suma += datos[i]' con condición: i == 4
// En el panel Variables ves: i=4, datos.length=5
// Pulsas F8 → i se vuelve 5 → ANTES de que crashee, ves el estado completo.
// La condición correcta es: i < datos.length   (sin el =)
```
