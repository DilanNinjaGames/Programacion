#### 🪲 "Rompe el Código" — Los 3 errores de compilación más comunes del pythonista

```java
// ══════════════════════════════════════════════════
// ERROR 1: Nombre de archivo ≠ nombre de clase
// ══════════════════════════════════════════════════

// Archivo guardado como: programa.java   ← minúscula
public class Programa {                   // ← mayúscula
    public static void main(String[] args) {}
}
// ❌ javac programa.java
// error: class Programa is public, should be declared in a file named Programa.java
//
// ✅ SOLUCIÓN: el archivo DEBE llamarse exactamente Programa.java


// ══════════════════════════════════════════════════
// ERROR 2: Firma de main() incorrecta
// ══════════════════════════════════════════════════

public class Demo {
    // ❌ Pythonista escribe main sin argumentos (como en Python)
    public static void main() {
        System.out.println("Hola");
    }
}
// ❌ $ java Demo
// Error: Main method not found in class Demo
// La JVM busca exactamente: main(String[] args)
// Aunque no uses args[], el parámetro es OBLIGATORIO en la firma.
//
// ✅ SOLUCIÓN:
// public static void main(String[] args) { ... }


// ══════════════════════════════════════════════════
// ERROR 3: Ejecutar el .class con extensión
// ══════════════════════════════════════════════════

// ❌ Error clásico (Python te acostumbró a incluir la extensión):
// $ java HolaMundo.class
// Error: Could not find or load main class HolaMundo.class
// Caused by: java.lang.ClassNotFoundException: HolaMundo.class
//
// ✅ SOLUCIÓN: java recibe el NOMBRE DE LA CLASE, no el archivo
// $ java HolaMundo              ← sin .class
```
