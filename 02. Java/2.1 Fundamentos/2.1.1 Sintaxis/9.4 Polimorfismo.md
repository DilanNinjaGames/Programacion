## 🔀 Polimorfismo — Overloading y Overriding

### 🔄 Contraste de Sintaxis

Java tiene dos tipos de polimorfismo con momentos de resolución distintos:

|Tipo|Nombre técnico|Se resuelve en|Mecanismo|
|---|---|---|---|
|**Overloading**|Polimorfismo estático|**Compilación**|Mismo nombre, diferente firma|
|**Overriding**|Polimorfismo dinámico|**Runtime (JVM)**|Subclase redefine método del padre|

**Overriding vs Overloading — diferencia visual:**

```java
// OVERLOADING: mismo nombre, diferente firma — misma clase o subclase
public void mover(int x) { ... }
public void mover(int x, int y) { ... }      // diferente nº de parámetros
public void mover(double x) { ... }           // diferente tipo

// OVERRIDING: misma firma exacta — subclase redefine el método del padre
class Padre  { public String saludar() { return "Hola soy Padre"; } }
class Hijo   { @Override
               public String saludar() { return "Hola soy Hijo";  } }
// Misma firma: public String saludar() — diferente implementación
```

**Dynamic Dispatch (Despacho Dinámico):** Cuando una referencia de tipo padre apunta a un objeto hijo, Java en runtime usa el tipo **real** del objeto para decidir qué método ejecutar. Esto es el corazón del polimorfismo dinámico.

### 💻 Bloque de Código — Polimorfismo dinámico en sistema de notificaciones

```java
// Sistema de notificaciones que demuestra dynamic dispatch

// ── CLASE BASE ─────────────────────────────────────────────────────────────
abstract class Notificacion {

    protected String destinatario;
    protected String mensaje;

    public Notificacion(String destinatario, String mensaje) {
        this.destinatario = destinatario;
        this.mensaje      = mensaje;
    }

    // Método que TODAS las subclases DEBEN implementar a su manera
    public abstract void enviar();

    // Método concreto compartido
    public void registrarEnvio() {
        System.out.println("[LOG] Notificación enviada a: " + destinatario);
    }

    // OVERLOADING dentro de la misma clase: método con variantes de firma
    public String formatear() {
        return "[" + destinatario + "] " + mensaje;
    }

    public String formatear(String prefijo) {            // distinto nº parámetros
        return prefijo + " " + formatear();
    }

    public String formatear(String prefijo, boolean mayus) {  // distinto nº parámetros
        String base = formatear(prefijo);
        return mayus ? base.toUpperCase() : base;
    }
}

// ── SUBCLASES con OVERRIDING ───────────────────────────────────────────────
class NotificacionEmail extends Notificacion {

    private String asunto;

    public NotificacionEmail(String email, String asunto, String cuerpo) {
        super(email, cuerpo);
        this.asunto = asunto;
    }

    @Override
    public void enviar() {
        System.out.printf("📧 EMAIL → %s | Asunto: '%s' | %s%n",
                          destinatario, asunto, mensaje);
        registrarEnvio();   // reutiliza método del padre
    }
}

class NotificacionSMS extends Notificacion {

    public NotificacionSMS(String telefono, String texto) {
        super(telefono, texto.length() > 160 ? texto.substring(0, 157) + "..." : texto);
    }

    @Override
    public void enviar() {
        System.out.printf("📱 SMS → %s | '%s' (%d chars)%n",
                          destinatario, mensaje, mensaje.length());
        registrarEnvio();
    }
}

class NotificacionPush extends Notificacion {

    private String appId;

    public NotificacionPush(String userId, String appId, String texto) {
        super(userId, texto);
        this.appId = appId;
    }

    @Override
    public void enviar() {
        System.out.printf("🔔 PUSH → User:%s | App:%s | '%s'%n",
                          destinatario, appId, mensaje);
        registrarEnvio();
    }
}

// ── DEMO: el poder del polimorfismo ───────────────────────────────────────
class DemoPolimorfismo {
    public static void main(String[] args) {

        // Array de tipo PADRE que contiene objetos HIJOS distintos
        // Esto es posible porque Email/SMS/Push IS-A Notificacion
        Notificacion[] notificaciones = {
            new NotificacionEmail("ana@mail.com", "Bienvenida", "Hola Ana!"),
            new NotificacionSMS("+52-555-1234", "Tu código OTP es: 847291"),
            new NotificacionPush("user_123", "mi.app", "Tienes 3 mensajes nuevos"),
            new NotificacionEmail("luis@mail.com", "Alerta", "Acceso sospechoso"),
        };

        // Dynamic dispatch: Java en runtime llama al enviar() correcto
        // El código no sabe ni le importa el tipo concreto — solo llama enviar()
        System.out.println("=== Enviando todas las notificaciones ===");
        for (Notificacion n : notificaciones) {
            n.enviar();   // ← la JVM decide cuál método usar en tiempo de ejecución
            System.out.println();
        }

        // OVERLOADING: el compilador elige la versión correcta en compilación
        Notificacion n = new NotificacionEmail("test@mail.com", "Test", "Prueba");
        System.out.println(n.formatear());                    // versión 0 args
        System.out.println(n.formatear(">>>"));               // versión 1 arg
        System.out.println(n.formatear(">>>", true));         // versión 2 args

        // Casting: convertir referencia de padre a tipo hijo concreto
        Notificacion ref = new NotificacionSMS("+1-800-0000", "Hola");
        if (ref instanceof NotificacionSMS) {           // verificar antes de castear
            NotificacionSMS sms = (NotificacionSMS) ref;    // downcast explícito
            sms.enviar();
        }

        // Java 16+ pattern matching for instanceof — más conciso
        if (ref instanceof NotificacionSMS sms) {   // declara y castea en una línea
            sms.enviar();
        }
    }
}
```

### 🪲 "Rompe el Código"

```java
// ❌ ERROR 1: confundir overloading con overriding

class Padre {
    public void mostrar(int x) {
        System.out.println("Padre int: " + x);
    }
}

class Hijo extends Padre {
    // El pythonista cree que está sobreescribiendo, pero ESTÁ SOBRECARGANDO:
    public void mostrar(double x) {   // diferente tipo → overloading, no overriding
        System.out.println("Hijo double: " + x);
    }
}

class Test {
    public static void main(String[] args) {
        Padre ref = new Hijo();
        ref.mostrar(5);     // ← llama a Padre.mostrar(int) ← ¡sorpresa!
        // La JVM busca overriding (misma firma) — no lo encuentra en Hijo
        // Cae al método del Padre
    }
}

// ✅ SOLUCIÓN: para overriding, la firma debe ser IDÉNTICA
class HijoCorregido extends Padre {
    @Override
    public void mostrar(int x) {      // misma firma → overriding real
        System.out.println("Hijo int: " + x);
    }
}


// ❌ ERROR 2: reducir visibilidad al hacer override (no permitido en Java)

class Base {
    public void metodo() { }    // public
}

class Derivada extends Base {
    @Override
    protected void metodo() { }   // ❌ attempting to assign weaker access privileges
    // No puedes hacer un método más restrictivo al sobreescribirlo
    // Un Derivada ES-UN Base → debe poder usarse como Base → public debe mantenerse

    // ✅ Puedes mantener o AMPLIAR el acceso, nunca reducirlo
    @Override
    public void metodo() { }    // ✅ mismo nivel de acceso
}
```
