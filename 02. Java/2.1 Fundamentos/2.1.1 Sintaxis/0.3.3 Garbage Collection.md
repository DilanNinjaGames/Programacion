#### 🔄 Contraste de Sintaxis

Python tiene GC (basado en conteo de referencias + detector de ciclos). Java también, pero con una diferencia importante de modelo mental:

|Aspecto|Python|Java|
|---|---|---|
|Algoritmo base|Conteo de referencias + cyclic GC|Mark-and-Sweep generacional|
|Cuándo se libera|Casi inmediato al llegar a 0 referencias|No determinístico — decide la JVM|
|Control manual|`del obj`, `gc.collect()`|No existe — **nunca llamas al GC**|
|Memoria que gestiona|Todo|Solo el **Heap** (Stack se libera solo)|
|Generaciones|No|Young Gen → Old Gen → Metaspace|

**El modelo generacional de Java** — por qué importa:

```
HEAP dividido en generaciones:
┌─────────────────────────────────────────────┐
│  Young Generation  (objetos nuevos)         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │  Eden    │  │Survivor 0│  │Survivor 1│  │
│  └──────────┘  └──────────┘  └──────────┘  │
│          Minor GC (frecuente, rápido)       │
├─────────────────────────────────────────────┤
│  Old Generation  (objetos longevos)         │
│          Major GC / Full GC (lento)         │
├─────────────────────────────────────────────┤
│  Metaspace  (metadata de clases)            │
└─────────────────────────────────────────────┘
```

La mayoría de objetos muere joven (variables locales temporales). El GC los limpia rápido en Young Gen. Solo los objetos que sobreviven muchos ciclos suben a Old Gen.

#### 💻 Bloque de Código — Observar el GC en acción

java

```java
// GarbageCollectionDemo.java
// Demuestra cuándo un objeto se vuelve elegible para el GC

public class GarbageCollectionDemo {

    // El GC llama a finalize() justo antes de destruir el objeto
    // NOTA: finalize() está deprecated en Java 9+ — solo para demostración pedagógica
    @Override
    protected void finalize() {
        System.out.println("GC destruyó: " + this.getClass().getSimpleName());
    }

    public static void crearObjectosTemporales() {
        // Estos objetos se crean en el Heap
        GarbageCollectionDemo obj1 = new GarbageCollectionDemo();
        GarbageCollectionDemo obj2 = new GarbageCollectionDemo();

        // obj1 pierde su referencia → elegible para GC
        obj1 = null;   // equivale a del obj1 en Python, pero el GC decide CUÁNDO

        // obj2 sigue siendo referenciado → NO es elegible todavía
        System.out.println("obj2 sigue vivo: " + obj2);

    } // Al salir del método: obj2 pierde su referencia del Stack → elegible para GC

    public static void main(String[] args) {
        crearObjectosTemporales();

        // Sugerimos al GC que corra (NO es una orden — la JVM puede ignorarlo)
        System.gc();   // en Python: gc.collect() — misma idea, mismo aviso

        // Memoria actual del Heap (útil para monitoreo)
        Runtime rt = Runtime.getRuntime();
        long usada  = (rt.totalMemory() - rt.freeMemory()) / 1024;
        long maxima = rt.maxMemory() / 1024;
        System.out.printf("Heap usado: %d KB / %d KB máx%n", usada, maxima);
    }
}
```

#### 🪲 "Rompe el Código"

java

```java
// ❌ ERROR: asumir que null libera memoria inmediatamente (como del en Python)

public class BugGC {
    public static void main(String[] args) {

        // Creamos objetos que acumulan mucha memoria
        byte[] bloque1 = new byte[50 * 1024 * 1024]; // 50 MB en el Heap
        byte[] bloque2 = new byte[50 * 1024 * 1024]; // otros 50 MB

        // El pythonista asume que esto libera 50 MB INMEDIATAMENTE:
        bloque1 = null;

        // ❌ Intenta crear otro bloque grande confiando en que ya se liberó:
        byte[] bloque3 = new byte[100 * 1024 * 1024]; // 100 MB — puede fallar
        // OutOfMemoryError: Java heap space
        // El GC no necesariamente corrió entre bloque1=null y esta línea

        // ✅ SOLUCIÓN — si necesitas forzar liberación antes de un objeto grande:
        bloque1 = null;
        bloque2 = null;
        System.gc();   // sugerencia — no garantía
        // En producción real: rediseña para no necesitar objetos tan grandes,
        // o usa -Xmx512m para aumentar el Heap máximo al lanzar la JVM:
        // $ java -Xmx512m MiPrograma
    }
}
```
