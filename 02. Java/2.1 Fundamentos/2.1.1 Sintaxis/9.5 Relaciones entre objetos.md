## 🔗 Relaciones entre Objetos — Asociación, Agregación y Composición

### 🔄 Contraste de Sintaxis

En Python las relaciones entre clases existen pero raramente se discuten formalmente. En Java (y en el diseño OOP en general) hay tres tipos con distintos grados de **dependencia del ciclo de vida**:

```
ASOCIACIÓN  →  "usa" o "conoce" — ciclos de vida INDEPENDIENTES
AGREGACIÓN  →  "tiene" (débil) — el hijo PUEDE existir sin el padre
COMPOSICIÓN →  "contiene" (fuerte) — el hijo NO existe sin el padre
```

**Visualización:**

```
Asociación:   Profesor ────────── Curso       (un profesor puede dar muchos cursos
                "enseña"                        y el curso existe sin ese profesor)

Agregación:   Universidad ◇──── Departamento  (el departamento puede existir
                "tiene"                         sin esa universidad específica)

Composición:  Casa ◆────── Habitación         (la habitación NO existe
                "contiene"                      fuera de esa casa)
```

### 💻 Bloque de Código — Los tres tipos de relación en un sistema universitario

```java
import java.util.ArrayList;
import java.util.List;

// ══════════════════════════════════════════════════════════════════════════
// ASOCIACIÓN: Estudiante "conoce" / "toma" Cursos
// Ciclos de vida completamente independientes
// ══════════════════════════════════════════════════════════════════════════

class Curso {
    private String codigo;
    private String nombre;
    private int    creditos;

    public Curso(String codigo, String nombre, int creditos) {
        this.codigo   = codigo;
        this.nombre   = nombre;
        this.creditos = creditos;
    }

    public String getCodigo()  { return codigo;  }
    public String getNombre()  { return nombre;  }
    public int    getCreditos(){ return creditos; }

    @Override
    public String toString() {
        return String.format("Curso[%s: %s, %d créditos]", codigo, nombre, creditos);
    }
}

class Estudiante {
    private String nombre;
    private String matricula;
    // ASOCIACIÓN: el Estudiante tiene una lista de Cursos
    // Los cursos existen independientemente del estudiante
    private List<Curso> cursosInscritos;

    public Estudiante(String nombre, String matricula) {
        this.nombre          = nombre;
        this.matricula       = matricula;
        this.cursosInscritos = new ArrayList<>();
    }

    // La ASOCIACIÓN se materializa en este método: un Curso externo se vincula
    public void inscribirCurso(Curso curso) {
        cursosInscritos.add(curso);   // el estudiante REFERENCIA el curso, no lo crea
    }

    public void mostrarCursos() {
        System.out.println(nombre + " (" + matricula + ") está inscrito en:");
        for (Curso c : cursosInscritos) {
            System.out.println("  " + c);
        }
    }
}

// ══════════════════════════════════════════════════════════════════════════
// AGREGACIÓN: Universidad "tiene" Departamentos (relación débil)
// Un Departamento puede pertenecer a otra universidad o existir independiente
// ══════════════════════════════════════════════════════════════════════════

class Departamento {
    private String nombre;
    private String jefe;

    public Departamento(String nombre, String jefe) {
        this.nombre = nombre;
        this.jefe   = jefe;
    }

    @Override
    public String toString() {
        return "Depto[" + nombre + ", jefe: " + jefe + "]";
    }
}

class Universidad {
    private String nombre;
    // AGREGACIÓN: la Universidad TIENE departamentos
    // Los departamentos se crean FUERA y se pasan — podrían "sobrevivir" a la universidad
    private List<Departamento> departamentos;

    public Universidad(String nombre) {
        this.nombre        = nombre;
        this.departamentos = new ArrayList<>();
    }

    // El Departamento se crea FUERA y se agrega — ciclo de vida separado
    public void agregarDepartamento(Departamento dept) {
        departamentos.add(dept);   // agrega referencia — no crea el objeto
    }

    public void mostrarEstructura() {
        System.out.println("Universidad: " + nombre);
        for (Departamento d : departamentos) {
            System.out.println("  └─ " + d);
        }
    }
}

// ══════════════════════════════════════════════════════════════════════════
// COMPOSICIÓN: Contrato "contiene" Cláusulas (relación fuerte)
// Las Cláusulas NO tienen sentido fuera del Contrato — ciclo de vida dependiente
// ══════════════════════════════════════════════════════════════════════════

class Clausula {
    private int    numero;
    private String descripcion;

    // El constructor es package-private (sin modificador) para que solo
    // Contrato pueda crear Cláusulas — refuerza la composición
    Clausula(int numero, String descripcion) {
        this.numero      = numero;
        this.descripcion = descripcion;
    }

    @Override
    public String toString() {
        return String.format("  Cláusula %d: %s", numero, descripcion);
    }
}

class Contrato {
    private String titular;
    private String fecha;
    // COMPOSICIÓN: Contrato CREA y POSEE las Cláusulas
    // Las cláusulas se crean DENTRO del Contrato y no existen fuera de él
    private List<Clausula> clausulas;

    public Contrato(String titular, String fecha) {
        this.titular  = titular;
        this.fecha    = fecha;
        this.clausulas = new ArrayList<>();  // el contrato crea su propia lista
    }

    // El Contrato crea internamente las cláusulas — no se reciben desde fuera
    public void agregarClausula(String descripcion) {
        int numero = clausulas.size() + 1;
        clausulas.add(new Clausula(numero, descripcion));  // COMPOSICIÓN: new aquí
    }

    public void mostrarContrato() {
        System.out.println("═══ Contrato ═══");
        System.out.println("Titular : " + titular);
        System.out.println("Fecha   : " + fecha);
        System.out.println("Cláusulas:");
        for (Clausula c : clausulas) {
            System.out.println(c);
        }
        System.out.println("════════════════");
    }
}

// ── PROGRAMA PRINCIPAL ─────────────────────────────────────────────────────
class DemoRelaciones {
    public static void main(String[] args) {

        System.out.println("══ ASOCIACIÓN ══");
        Curso matematicas = new Curso("MAT101", "Matemáticas Discretas", 4);
        Curso programacion = new Curso("PRG201", "Algoritmos", 5);
        Curso bd           = new Curso("BD301",  "Bases de Datos", 4);

        Estudiante estudiante = new Estudiante("Ana García", "A2024001");
        estudiante.inscribirCurso(matematicas);   // cursos existen independientemente
        estudiante.inscribirCurso(programacion);
        estudiante.inscribirCurso(bd);
        estudiante.mostrarCursos();

        // Los cursos siguen vivos aunque eliminemos al estudiante
        System.out.println("Curso sin estudiante: " + matematicas);

        System.out.println("\n══ AGREGACIÓN ══");
        Departamento it     = new Departamento("Ingeniería TI", "Dr. López");
        Departamento ciencias = new Departamento("Ciencias Básicas", "Dra. Martínez");

        Universidad uni = new Universidad("Universidad Tecnológica");
        uni.agregarDepartamento(it);
        uni.agregarDepartamento(ciencias);
        uni.mostrarEstructura();

        // El departamento puede reasignarse a otra universidad (ciclo independiente)
        Universidad uni2 = new Universidad("Instituto de Postgrado");
        uni2.agregarDepartamento(it);  // mismo objeto, diferente universidad
        uni2.mostrarEstructura();

        System.out.println("\n══ COMPOSICIÓN ══");
        Contrato contrato = new Contrato("Luis Pérez", "2025-01-15");
        contrato.agregarClausula("El servicio inicia el 1 de febrero de 2025");
        contrato.agregarClausula("El pago se realiza los días 5 de cada mes");
        contrato.agregarClausula("La duración del contrato es de 12 meses");
        contrato.mostrarContrato();

        // Las cláusulas NO existen fuera del contrato — composición real
        // No hay forma de crear un objeto Clausula desde afuera (package-private)
    }
}
```

### 🪲 "Rompe el Código"

```java
// ❌ ERROR CONCEPTUAL: implementar composición como agregación (bug de diseño)

// Versión INCORRECTA: la Habitación se crea fuera y se pasa — es AGREGACIÓN
class HabitacionMal {
    String tipo;
    HabitacionMal(String tipo) { this.tipo = tipo; }
}

class CasaMal {
    List<HabitacionMal> habitaciones = new ArrayList<>();

    // ❌ DISEÑO INCORRECTO para composición:
    // La habitación se crea fuera y se pasa — puede existir sin la casa
    public void agregarHabitacion(HabitacionMal h) {
        habitaciones.add(h);
    }
}

// Uso que rompe la semántica de composición:
HabitacionMal hab = new HabitacionMal("Sala");  // ← existe ANTES de la casa
CasaMal casa = new CasaMal();
casa.agregarHabitacion(hab);
// La habitación sobrevive si eliminamos la casa — eso NO es composición

// ✅ VERSIÓN CORRECTA: composición real
// La Casa CREA las habitaciones internamente — no pueden existir sin ella
class Casa {
    private List<Habitacion> habitaciones = new ArrayList<>();

    // La Casa controla la creación de sus partes
    public void construirHabitacion(String tipo) {
        habitaciones.add(new Habitacion(tipo));  // new aquí dentro — composición
    }

    // Habitacion como clase interna: refuerza que solo Casa la crea
    private class Habitacion {
        String tipo;
        Habitacion(String tipo) { this.tipo = tipo; }
        @Override public String toString() { return "Habitación[" + tipo + "]"; }
    }

    public void describir() {
        System.out.println("Casa con " + habitaciones.size() + " habitaciones:");
        habitaciones.forEach(h -> System.out.println("  " + h));
    }
}

// ✅ Uso correcto de composición:
Casa casaCorrecta = new Casa();
casaCorrecta.construirHabitacion("Sala");
casaCorrecta.construirHabitacion("Dormitorio");
casaCorrecta.construirHabitacion("Cocina");
casaCorrecta.describir();
// No hay forma de crear una Habitacion fuera de Casa — ciclo de vida dependiente ✅
```
