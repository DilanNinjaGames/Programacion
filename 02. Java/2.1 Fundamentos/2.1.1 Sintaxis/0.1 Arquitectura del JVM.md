La JVM tiene tres subsistemas que debes conocer porque afectan directamente el comportamiento de tu código:

```
Tu código .java
      ↓ javac
  Bytecode .class  (independiente del OS)
      ↓
┌─────────────────────────────────────────┐
│              JVM                        │
│                                         │
│  1. Class Loader    ← carga .class      │
│       ↓                                 │
│  2. Runtime Data Areas:                 │
│     ├── Stack    ← variables locales,   │
│     │             llamadas a métodos    │
│     └── Heap     ← objetos con new()   │
│       ↓                                 │
│  3. Execution Engine                    │
│     ├── Interpreter   (línea a línea)   │
│     └── JIT Compiler  (optimiza hot     │
│                        code en runtime) │
└─────────────────────────────────────────┘
      ↓
  OS / Hardware (Linux, Windows, macOS)
```

**Por qué importa el Stack vs Heap desde ya:**

- En Python no piensas en esto — el GC lo maneja todo.
- En Java: tipos **primitivos** (`int`, `double`, `boolean`…) viven en el **Stack**. Los **objetos** (`String`, tus clases, arrays) viven en el **Heap**. Esta distinción causará bugs reales cuando compares objetos con `==` (sección de POO).
