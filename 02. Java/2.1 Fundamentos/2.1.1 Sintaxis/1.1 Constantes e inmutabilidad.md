#### 🔄 Contraste de Sintaxis

En Python, las "constantes" son variables con nombre en UPPER_SNAKE_CASE — es una **convención**, no un mecanismo del lenguaje. Nada te impide reasignarlas. En Java, `final` es una **garantía del compilador**: si intentas reasignar una variable `final`, el código no compila.

```
Python (convención):        Java (garantía del compilador):
────────────────────        ────────────────────────────────
MAX_SIZE = 100              static final int MAX_SIZE = 100;
MAX_SIZE = 200  # ← ok      MAX_SIZE = 200;  // ← ERROR DE COMPILACIÓN
```

#### 💻 Bloque de Código — final en variables, parámetros y referencias

java

```java
// ConstantesDemo.java

public class ConstantesDemo {

    // ── CONSTANTE DE CLASE: static final ─────────────────────────────────
    // static: pertenece a la clase, no a instancias
    // final:  no puede reasignarse después de la inicialización
    // Por convención: UPPER_SNAKE_CASE
    static final double PI             = 3.14159265358979;
    static final int    MAX_INTENTOS   = 3;
    static final String MONEDA_DEFAULT = "USD";

    public static void main(String[] args) {

        // ── final EN VARIABLE LOCAL ───────────────────────────────────────
        final int puntosDePaso = 60;  // asignada una sola vez
        // puntosDePaso = 70;         // ❌ compilación: cannot assign a value to final variable

        // Útil para claridad intencional: "este valor no debe cambiar aquí"
        final double tasaImpuesto = 0.16;
        double precioNeto   = 100.0;
        double precioFinal  = precioNeto * (1 + tasaImpuesto); // tasaImpuesto no cambia
        System.out.printf("Precio final: $%.2f%n", precioFinal);

        // ── final EN ARRAYS (ojo: la referencia es final, no el contenido) ─
        final int[] calificaciones = {85, 90, 78, 92};
        calificaciones[0] = 100;     // ✅ modificar el CONTENIDO está permitido
        // calificaciones = new int[]{1,2,3}; // ❌ reasignar la REFERENCIA no

        // Esto confunde a los pythonistas: final en objeto ≠ objeto inmutable
        // final solo garantiza que la VARIABLE no apuntará a otro objeto
        System.out.println("Primera calificación: " + calificaciones[0]); // 100

        // ── CADENAS INMUTABLES: String en Java es inmutable como en Python ──
        String saludo = "Hola";
        saludo = saludo + ", mundo";  // NO modifica "Hola" — crea un nuevo String
        // El objeto "Hola" sigue en memoria hasta que el GC lo limpie
        // Para manipulación intensiva de strings → usar StringBuilder (mutable)

        StringBuilder sb = new StringBuilder("Hola");
        sb.append(", mundo");    // modifica el MISMO objeto en memoria
        sb.append("!");
        System.out.println(sb.toString());  // "Hola, mundo!"
    }

    // ── final EN PARÁMETROS ───────────────────────────────────────────────
    public static double calcularAreaCirculo(final double radio) {
        // radio = -1; // ❌ no permite modificar el parámetro dentro del método
        // Útil para documentar intención y prevenir bugs por modificación accidental
        return PI * radio * radio;
    }
}
```

#### 🪲 "Rompe el Código"

java

```java
// ❌ ERROR: pensar que final en un objeto lo hace completamente inmutable

import java.util.ArrayList;

public class BugFinal {
    public static void main(String[] args) {

        final ArrayList<String> lista = new ArrayList<>();

        lista.add("Ana");    // ✅ permitido — modifica el CONTENIDO del objeto
        lista.add("Luis");   // ✅ permitido

        // El pythonista asume que final = tuple (inmutable total) → INCORRECTO
        System.out.println(lista);  // [Ana, Luis] — el contenido cambió

        lista = new ArrayList<>();  // ❌ compilación: cannot assign to final variable
        // final solo protege la REFERENCIA (la variable no puede apuntar a otro objeto)
        // NO protege el estado interno del objeto
    }
}

// ✅ RESUMEN:
// final int x = 5          → x nunca puede ser otro valor
// final String s = "hola"  → s nunca puede apuntar a otro String
//                             (y String en sí es inmutable — doble garantía)
// final List<T> lista      → lista nunca puede apuntar a otra List
//                             PERO lista.add() sigue funcionando
// Para lista verdaderamente inmutable → Collections.unmodifiableList(lista)
```
