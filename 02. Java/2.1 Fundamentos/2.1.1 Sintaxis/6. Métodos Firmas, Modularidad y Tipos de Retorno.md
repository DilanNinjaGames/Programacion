#### 🔄 Contraste de Sintaxis

En Python, `def` crea una función con tipado opcional. En Java, cada método **debe** declarar explícitamente su firma completa:

```
Python:    def  calcular   (a,    b   ) -> int:
Java:   public  static  int calcular  (int a, int b) {
           ↑       ↑     ↑                ↑
        acceso  ámbito  retorno       tipos obligatorios
```

**Sobrecarga de métodos (Overloading)** — exclusivo de Java entre los dos lenguajes:

java

```java
// Mismo nombre, diferente firma → el compilador elige cuál usar
public static int    sumar(int a, int b)       { return a + b; }
public static double sumar(double a, double b) { return a + b; }
public static int    sumar(int a, int b, int c){ return a + b + c; }
```

#### 💻 Bloque de Código — Métodos con void, retorno, sobrecarga y varargs

java

```java
// MetodosDemo.java

public class MetodosDemo {

    // ── void: no retorna nada (equivale a def sin return en Python) ───────
    public static void imprimirLinea(char caracter, int cantidad) {
        for (int i = 0; i < cantidad; i++) {
            System.out.print(caracter);
        }
        System.out.println();
    }

    // ── Con retorno: el tipo declarado DEBE coincidir con lo retornado ────
    public static double calcularPromedio(int[] numeros) {
        if (numeros == null || numeros.length == 0) {
            return 0.0;   // retorno anticipado — Java permite múltiples return
        }
        int suma = 0;
        for (int n : numeros) suma += n;
        return (double) suma / numeros.length;  // cast para división real
    }

    // ── SOBRECARGA: mismo nombre, diferente número de parámetros ──────────
    public static int maximo(int a, int b) {
        return (a >= b) ? a : b;
    }

    public static int maximo(int a, int b, int c) {
        return maximo(maximo(a, b), c);  // reutiliza la versión de 2 args
    }

    // ── SOBRECARGA: mismo nombre, diferente tipo de parámetros ────────────
    public static double maximo(double a, double b) {
        return (a >= b) ? a : b;
    }

    // ── VARARGS: número variable de argumentos (equivalente a *args Python)
    // Solo puede ser el ÚLTIMO parámetro; dentro del método es un array
    public static int sumarTodos(String etiqueta, int... numeros) {
        int total = 0;
        for (int n : numeros) total += n;
        System.out.println(etiqueta + " = " + total);
        return total;
    }

    public static void main(String[] args) {

        imprimirLinea('=', 40);
        double prom = calcularPromedio(new int[]{85, 92, 78, 96, 88});
        System.out.printf("Promedio: %.2f%n", prom);

        imprimirLinea('-', 40);

        // El compilador elige la sobrecarga correcta según los tipos de args
        System.out.println(maximo(5, 3));           // int, int       → 5
        System.out.println(maximo(5, 3, 9));        // int, int, int  → 9
        System.out.println(maximo(3.14, 2.71));     // double, double → 3.14

        imprimirLinea('-', 40);

        sumarTodos("Suma A", 1, 2, 3);              // 6
        sumarTodos("Suma B", 10, 20, 30, 40);       // 100
        sumarTodos("Vacío");                         // 0 — cero args para varargs
    }
}
```

public static int[] minMax(int[] arr) {
    int[] copia = arr.clone();
    Arrays.sort(copia);
    return new int[]{copia[0], copia[copia.length - 1]};

### Mecanismo de Paso de Parámetros: Pass-by-Value

#### 🔄 Contraste de Sintaxis

Java es **siempre pass-by-value** sin excepción. La confusión surge porque para objetos, el "valor" que se copia es la **referencia (dirección en memoria)**, no el objeto:

```
PRIMITIVOS → se copia el VALOR:
  int x = 5;  →  método recibe copia del 5
                  modificar el parámetro NO afecta x

OBJETOS/ARRAYS → se copia la REFERENCIA:
  int[] arr = {1,2,3};  →  método recibe copia de la dirección
                            modificar arr[0] SÍ afecta el original
                            (apuntan al mismo Heap)
                            reasignar arr = new int[]{} NO afecta al original
```

#### 💻 Bloque de Código — Los cuatro casos de pass-by-value

java

```java
// PassByValueDemo.java

public class PassByValueDemo {

    // CASO 1: primitivo — la copia no afecta al original
    public static void doblarPrimitivo(int numero) {
        numero *= 2;  // modifica solo la copia local
        System.out.println("  Dentro: " + numero);  // 20
    }

    // CASO 2: array — modificar contenido SÍ afecta al original
    public static void doblarElementos(int[] arr) {
        for (int i = 0; i < arr.length; i++) {
            arr[i] *= 2;  // modifica el objeto real en el Heap
        }
    }

    // CASO 3: reasignar referencia NO afecta al original
    public static void intentarReasignar(int[] arr) {
        arr = new int[]{99, 99, 99};  // variable local apunta a nuevo objeto
        // el arr del caller sigue igual
    }

    // CASO 4: swap que no funciona (el clásico ejemplo educativo)
    public static void swapFallido(int a, int b) {
        int temp = a; a = b; b = temp;
        // intercambia las COPIAS — el caller no ve ningún cambio
    }

    public static void main(String[] args) {

        // CASO 1
        int x = 10;
        doblarPrimitivo(x);
        System.out.println("Fuera: " + x);   // 10 — sin cambio ✅

        // CASO 2
        int[] numeros = {1, 2, 3, 4, 5};
        doblarElementos(numeros);
        System.out.println(java.util.Arrays.toString(numeros));
        // [2, 4, 6, 8, 10] — SÍ cambió ✅

        // CASO 3
        int[] original = {1, 2, 3};
        intentarReasignar(original);
        System.out.println(java.util.Arrays.toString(original));
        // [1, 2, 3] — sin cambio ✅ (reasignación local no se propaga)

        // CASO 4
        int p = 5, q = 10;
        swapFallido(p, q);
        System.out.println("p=" + p + " q=" + q);  // p=5 q=10 — sin intercambio

        // ✅ El swap real se hace directamente en el contexto del caller:
        int temp = p; p = q; q = temp;
        System.out.println("p=" + p + " q=" + q);  // p=10 q=5 ✅
    }
}
```

// ❌ ERROR: creer que pasar un array permite "agregarle elementos" desde un método

public static void agregarElemento(int[] arr, int valor) {
    arr = Arrays.copyOf(arr, arr.length + 1);  // crea NUEVO array local
    arr[arr.length - 1] = valor;
    // 'arr' local apunta al nuevo array — el del caller no cambió
}

public static void main(String[] args) {
    int[] miArray = {1, 2, 3};
    agregarElemento(miArray, 99);
    System.out.println(miArray.length);  // ❌ sigue siendo 3, no 4

    // ✅ SOLUCIÓN: retornar el nuevo array y reasignar en el caller
    miArray = agregarYRetornar(miArray, 99);
    System.out.println(miArray.length);  // ✅ ahora sí es 4
}

public static int[] agregarYRetornar(int[] arr, int valor) {
    int[] nuevo = Arrays.copyOf(arr, arr.length + 1);
    nuevo[nuevo.length - 1] = valor;
    return nuevo;  // el caller actualiza su propia referencia
}
