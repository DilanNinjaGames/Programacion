#### 🔄 Contraste de Sintaxis

El `if-else` en Java es casi idéntico a Python en lógica, con tres diferencias de forma: **paréntesis obligatorios** en la condición, **llaves `{}`** para los bloques, y `else if` en lugar de `elif`.

El `switch` es donde la diferencia es más profunda. Python no tuvo `switch` hasta `match/case` en 3.10. Java tiene `switch` desde su origen y en Java 14+ introdujo una versión mejorada con sintaxis de flecha `->` que elimina la trampa del **fall-through**.

|Característica|Python|Java (clásico)|Java 14+ (moderno)|
|---|---|---|---|
|Selección múltiple|`if/elif/else`|`if/else if/else`|igual|
|Switch|`match/case` (3.10+)|`switch` con `break`|`switch` con `->`|
|Fall-through|No aplica|Sí, por defecto ⚠️|No, por defecto ✅|
|Switch como expresión|No|No|Sí, retorna valor|

java

```java
// ── if / else if / else ───────────────────────────────────────────────────
// Python: if / elif / else
// Java:   if / else if / else  ← dos palabras, no una

int nota = 85;

if (nota >= 90) {           // paréntesis obligatorios
    System.out.println("A");
} else if (nota >= 80) {    // 'else if', no 'elif'
    System.out.println("B");
} else if (nota >= 70) {
    System.out.println("C");
} else {
    System.out.println("F");
}
```

#### 💻 Bloque de Código — `switch` clásico vs moderno

java

```java
// SwitchDemo.java
// Muestra la trampa del fall-through y cómo la resuelve el switch moderno

public class SwitchDemo {

    public static void main(String[] args) {

        // ── SWITCH CLÁSICO (Java 1 en adelante) ───────────────────────────
        // La trampa: si olvidas 'break', el código "cae" al siguiente caso
        int diaSemana = 3;
        String nombreDia;

        switch (diaSemana) {
            case 1:
                nombreDia = "Lunes";
                break;          // ← OBLIGATORIO para detener el fall-through
            case 2:
                nombreDia = "Martes";
                break;
            case 3:
                nombreDia = "Miércoles";
                break;
            case 4:
            case 5:             // ← fall-through INTENCIONAL: 4 y 5 comparten código
                nombreDia = "Jueves o Viernes";
                break;
            default:            // equivale al 'else' — siempre incluirlo
                nombreDia = "Fin de semana";
        }
        System.out.println(nombreDia);  // "Miércoles"

        // ── SWITCH MODERNO con -> (Java 14+) ──────────────────────────────
        // Sin fall-through, sin break, puede usarse como expresión
        String tipo = switch (diaSemana) {
            case 1, 2, 3, 4, 5 -> "Día laboral";   // múltiples casos con ,
            case 6, 7           -> "Fin de semana";
            default             -> "Día inválido";
        };  // ← termina con ; porque es una expresión asignada
        System.out.println(tipo);  // "Día laboral"

        // ── SWITCH con String (Java 7+) ────────────────────────────────────
        // Python permite match sobre cualquier tipo — Java switch acepta:
        // int, byte, short, char, String, enums (NO double, long, boolean)
        String estacion = "verano";

        String descripcion = switch (estacion.toLowerCase()) {
            case "primavera" -> "Flores y lluvia";
            case "verano"    -> "Calor intenso";
            case "otoño"     -> "Hojas caídas";
            case "invierno"  -> "Frío y nieve";
            default          -> "Estación desconocida";
        };
        System.out.println(descripcion);  // "Calor intenso"

        // ── SWITCH con yield (bloque de múltiples líneas) ─────────────────
        // Cuando el case necesita más de una sentencia, usa yield para retornar
        int mes = 2;
        int diasEnMes = switch (mes) {
            case 1, 3, 5, 7, 8, 10, 12 -> 31;
            case 4, 6, 9, 11            -> 30;
            case 2 -> {
                // bloque con lógica — yield retorna el valor del bloque
                int anio = 2024;
                boolean esBisiesto = (anio % 4 == 0 && anio % 100 != 0)
                                  || (anio % 400 == 0);
                yield esBisiesto ? 29 : 28;   // yield en lugar de return
            }
            default -> throw new IllegalArgumentException("Mes inválido: " + mes);
        };
        System.out.println("Días en mes " + mes + ": " + diasEnMes);  // 29
    }
}
```

#### 🪲 "Rompe el Código"

```java
// ❌ ERROR CLÁSICO: olvidar 'break' en switch clásico (fall-through no deseado)

public class BugFallThrough {
    public static void main(String[] args) {

        int nivel = 2;
        String permiso = "";

        switch (nivel) {
            case 1:
                permiso = "lectura";
                // ❌ sin break — cae al siguiente caso
            case 2:
                permiso = "escritura";
                // ❌ sin break — cae al siguiente caso
            case 3:
                permiso = "administrador";
                // ❌ sin break — cae al default
            default:
                permiso = "sin permiso";
        }

        System.out.println(permiso);  // "sin permiso" ← aunque nivel=2
        // El código ejecutó TODOS los casos desde el 2 hasta default
        // Último en asignarse: "sin permiso"

        // ✅ SOLUCIÓN 1 — agregar break en cada case del switch clásico
        // ✅ SOLUCIÓN 2 — usar switch moderno con -> (Java 14+):
        String permisoCorrect = switch (nivel) {
            case 1  -> "lectura";
            case 2  -> "escritura";    // ← se detiene aquí automáticamente
            case 3  -> "administrador";
            default -> "sin permiso";
        };
        System.out.println(permisoCorrect);  // "escritura" ✅
    }
}
```
