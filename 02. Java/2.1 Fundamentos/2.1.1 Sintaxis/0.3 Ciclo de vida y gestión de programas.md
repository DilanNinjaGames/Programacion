## ⚙️ Ciclo de Vida y Gestión del Programa

### Fases de Compilación

#### 🔄 Contraste de Sintaxis

En Python, cuando ejecutas `python3 script.py`, el intérprete hace todo esto **invisible e instantáneamente**. Java lo hace **explícito y en dos comandos separados**. Las fases internas del compilador `javac` son:

```
Tu archivo .java
       ↓
┌──────────────────────────────────────────┐
│           javac (compilador)             │
│                                          │
│  Fase 1 — Analizador LÉXICO             │
│  "Tokenización": convierte el texto en  │
│  tokens: palabras clave, identificadores│
│  literales, operadores.                 │
│  Ej: 'int x = 5;' → [int][x][=][5][;] │
│                ↓                        │
│  Fase 2 — Analizador SINTÁCTICO         │
│  Verifica que los tokens forman         │
│  estructuras válidas (AST).             │
│  Detecta: llaves sin cerrar, ; faltante │
│                ↓                        │
│  Fase 3 — Analizador SEMÁNTICO          │
│  Verifica SIGNIFICADO: tipos correctos, │
│  variables declaradas, tipos compatibles│
│  Detecta: int x = "hola" ← error tipo  │
└──────────────────────────────────────────┘
       ↓
   archivo .class (bytecode)
```

**Lo que Python detecta en runtime, Java lo detecta en compilación.** Esto significa que en Java, si asignas un `String` a un `int`, el error aparece **antes de ejecutar una sola línea**. En Python lo descubrirías cuando esa línea se ejecuta.

### Edición, Compilación y Ejecución

#### 🔄 Contraste de Sintaxis

```
PYTHON:                          JAVA:
─────────────────                ──────────────────────────────
$ nano script.py                 $ nano HolaMundo.java
$ python3 script.py   (1 paso)   $ javac HolaMundo.java  (paso 1: compila)
                                 → genera HolaMundo.class
                                 $ java HolaMundo        (paso 2: ejecuta)
```

**Reglas de nomenclatura que no son opcionales en Java:**

- El nombre del **archivo** `.java` debe ser **idéntico** al nombre de la **clase pública** que contiene.
- `HolaMundo.java` → debe contener `public class HolaMundo { ... }`
- Si el nombre no coincide: **error de compilación inmediato**.
- En Python puedes llamar tu archivo como quieras — en Java no.

### Estructura Básica y el Método `main()`

#### 🔄 Contraste de Sintaxis

En Python, el punto de entrada es el archivo en sí. La convención `if __name__ == "__main__"` es **opcional**. En Java, el punto de entrada es **obligatorio y tiene una firma exacta e inmutable**:

```java
public static void main(String[] args)
//  ↑      ↑     ↑    ↑      ↑
//  │      │     │    │      └─ array de Strings: argumentos de línea de comandos
//  │      │     │    └──────── nombre obligatorio: exactamente "main"
//  │      │     └───────────── no retorna nada (void ≠ None)
//  │      └─────────────────── pertenece a la clase, no a una instancia
//  └────────────────────────── visible para la JVM desde fuera
```

Si cambias **cualquier** parte de esta firma, la JVM no encontrará el punto de entrada y lanzará `Main method not found`.

**Comparación directa Python → Java:**

```python
# Python — punto de entrada implícito
# archivo: hola_mundo.py

def saludar(nombre):
    print(f"Hola, {nombre}")

if __name__ == "__main__":   # opcional
    saludar("Python")
```

```java
// Java — punto de entrada explícito y obligatorio
// archivo: HolaMundo.java  ← nombre idéntico a la clase pública

public class HolaMundo {                        // clase envolvente obligatoria

    // Método auxiliar — equivale a def saludar(nombre): en Python
    public static void saludar(String nombre) { // tipo de parámetro obligatorio
        System.out.println("Hola, " + nombre);  // no hay f-strings aquí
    }

    // Punto de entrada — la JVM busca exactamente este método
    public static void main(String[] args) {
        saludar("Java");                         // llamada igual que en Python
    }
}
```
#### 💻 Bloque de Código — Programa con argumentos de línea de comandos

```java
// Calculadora.java
// Demuestra: estructura de clase, main(), args[] y manejo básico

public class Calculadora {

    /**
     * Suma dos números enteros.
     * En Python: def sumar(a, b): return a + b
     * En Java: los tipos de parámetros y retorno son OBLIGATORIOS.
     *
     * @param a primer operando
     * @param b segundo operando
     * @return resultado de la suma
     */
    public static int sumar(int a, int b) {
        return a + b;   // 'return' igual que Python, pero el tipo ya está declarado
    }

    public static void main(String[] args) {
        // args[] captura argumentos de terminal: $ java Calculadora 10 20
        // En Python: sys.argv[1], sys.argv[2]
        // En Java:   args[0],     args[1]

        // Verificar que el usuario pasó exactamente 2 argumentos
        if (args.length != 2) {
            // System.err → canal de error estándar (stderr), como en Python
            System.err.println("Uso: java Calculadora <num1> <num2>");
            System.exit(1);   // código de salida 1 = error — igual que sys.exit(1)
        }

        // Integer.parseInt() convierte String → int
        // En Python: int("10") — misma idea, diferente sintaxis
        int numero1 = Integer.parseInt(args[0]);
        int numero2 = Integer.parseInt(args[1]);

        // Llamada al método auxiliar — mismo patrón que Python
        int resultado = sumar(numero1, numero2);

        // Concatenación con + — en Java el + entre String e int convierte el int
        // automáticamente a String (esto se llama "string concatenation")
        System.out.println("Resultado: " + resultado);

        // Alternativa moderna con formatted strings (Java 15+):
        // System.out.println("Resultado: %d".formatted(resultado));
    }
}
```

```bash
# Compilar y ejecutar:
$ javac Calculadora.java        # genera Calculadora.class
$ java Calculadora 10 20        # Resultado: 30
$ java Calculadora              # Uso: java Calculadora <num1> <num2>
```
