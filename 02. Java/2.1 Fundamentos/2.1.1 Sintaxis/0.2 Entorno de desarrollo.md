#### 💻 Bloque de Código — Verificar tu entorno

```bash
# En terminal — verifica qué tienes instalado
javac --version    # Si responde: tienes JDK ✅
java  --version    # Si responde: tienes JRE/JDK ✅

# javac = compilador (solo en JDK)
# java  = ejecutor de bytecode (JVM, en JRE y JDK)
```

```java
// VerificarEntorno.java
// Este programa imprime información real de la JVM que lo ejecuta

public class VerificarEntorno {
    public static void main(String[] args) {

        // System.getProperty() consulta propiedades internas de la JVM
        String versionJava = System.getProperty("java.version");
        String vendorJVM   = System.getProperty("java.vm.name");
        String sistemaOS   = System.getProperty("os.name");

        // En Python sería: import platform; platform.system()
        // En Java, la JVM abstrae el OS — tu código no cambia entre plataformas
        System.out.println("Versión Java : " + versionJava);
        System.out.println("JVM usada    : " + vendorJVM);
        System.out.println("Sistema OS   : " + sistemaOS);

        // Runtime.getRuntime() da acceso a la JVM en ejecución
        Runtime jvm = Runtime.getRuntime();

        // Memoria disponible para objetos en el Heap (en bytes)
        long memoriaHeap = jvm.maxMemory();
        System.out.println("Heap máximo  : " + (memoriaHeap / 1024 / 1024) + " MB");

        // CPUs disponibles para la JVM (útil para concurrencia)
        System.out.println("Núcleos CPU  : " + jvm.availableProcessors());
    }
}
```

```java
// ❌ ERROR CLÁSICO: confundir JRE con JDK en el PATH

// Situación: instalaste solo el JRE (para ejecutar), no el JDK (para desarrollar)
// Intentas compilar desde terminal:

// $ javac HolaMundo.java
// bash: javac: command not found   ← javac no existe en JRE

// ✅ POR QUÉ OCURRE:
// El JRE solo incluye 'java' (para ejecutar .class ya compilados).
// 'javac' es el COMPILADOR — solo existe en el JDK.
// Un pythonista lo ve raro porque 'python3' hace ambas cosas.

// ✅ SOLUCIÓN:
// 1. Instalar JDK (no JRE): https://adoptium.net
// 2. Verificar que JAVA_HOME apunta al JDK, no al JRE
// $ echo $JAVA_HOME          → /usr/lib/jvm/java-21-openjdk   ✅
// $ which javac               → /usr/lib/jvm/java-21-openjdk/bin/javac ✅
```
