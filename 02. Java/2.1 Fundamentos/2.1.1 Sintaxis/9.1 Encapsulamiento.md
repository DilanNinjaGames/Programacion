## 🔒 Encapsulamiento — Modificadores, Getters y Setters

### 🔄 Contraste de Sintaxis

En Python el encapsulamiento es **convencional**: `_privado` (un guión) sugiere "no usar fuera", `__muy_privado` (doble guión) activa name mangling — pero nada lo impide realmente. En Java es **reforzado por el compilador** mediante modificadores de acceso.

|Modificador|Clase|Paquete|Subclase|Mundo|
|---|---|---|---|---|
|`private`|✅|❌|❌|❌|
|_(sin modificador)_|✅|✅|❌|❌|
|`protected`|✅|✅|✅|❌|
|`public`|✅|✅|✅|✅|

**Regla de oro:** atributos `private`, métodos `public`. Los getters/setters son el puente controlado.

**Convención JavaBeans** (estándar para getters/setters):

```java
// Para atributo:  private Tipo nombreAtributo
// Getter:         public Tipo getNombreAtributo()       // boolean: isNombreAtributo()
// Setter:         public void setNombreAtributo(Tipo x)
```

### 💻 Bloque de Código — Encapsulamiento con validación en setters

```java
// Persona.java
// Encapsulamiento real: los setters validan antes de asignar

public class Persona {

    // Todos los atributos PRIVADOS — nadie los toca directamente desde afuera
    private String nombre;
    private int    edad;
    private String email;
    private double salario;

    // Constructor con validación
    public Persona(String nombre, int edad, String email, double salario) {
        // Reutilizamos los setters para centralizar la validación
        setNombre(nombre);
        setEdad(edad);
        setEmail(email);
        setSalario(salario);
    }

    // ── GETTERS ───────────────────────────────────────────────────────────
    public String getNombre() { return nombre; }
    public int    getEdad()   { return edad;   }
    public String getEmail()  { return email;  }
    public double getSalario(){ return salario; }

    // ── SETTERS CON VALIDACIÓN ────────────────────────────────────────────
    // El encapsulamiento real está AQUÍ: controlamos qué valores se aceptan

    public void setNombre(String nombre) {
        if (nombre == null || nombre.trim().isEmpty()) {
            throw new IllegalArgumentException("Nombre no puede estar vacío");
        }
        // trim() elimina espacios en los extremos — equivale a str.strip() en Python
        this.nombre = nombre.trim();
    }

    public void setEdad(int edad) {
        if (edad < 0 || edad > 150) {
            throw new IllegalArgumentException("Edad inválida: " + edad);
        }
        this.edad = edad;
    }

    public void setEmail(String email) {
        if (email == null || !email.contains("@")) {
            throw new IllegalArgumentException("Email inválido: " + email);
        }
        this.email = email.toLowerCase();  // normalizar a minúsculas
    }

    public void setSalario(double salario) {
        if (salario < 0) {
            throw new IllegalArgumentException("Salario no puede ser negativo");
        }
        this.salario = salario;
    }

    // Método de negocio: aumento con límite
    public void aplicarAumento(double porcentaje) {
        if (porcentaje < 0 || porcentaje > 100) {
            throw new IllegalArgumentException("Porcentaje inválido: " + porcentaje);
        }
        this.salario *= (1 + porcentaje / 100.0);
    }

    @Override
    public String toString() {
        return String.format("Persona{nombre='%s', edad=%d, email='%s', salario=$%.2f}",
                             nombre, edad, email, salario);
    }

    public static void main(String[] args) {

        Persona p = new Persona("Ana García", 30, "ANA@ejemplo.com", 50000.0);
        System.out.println(p);
        // Persona{nombre='Ana García', edad=30, email='ana@ejemplo.com', salario=$50000.00}

        p.aplicarAumento(10);
        System.out.println("Nuevo salario: $" + p.getSalario());  // 55000.0

        // ✅ El atributo privado es inaccesible desde fuera:
        // p.salario = -999;  // ❌ salario has private access in Persona

        // La validación del setter protege la integridad:
        try {
            p.setEdad(-5);
        } catch (IllegalArgumentException e) {
            System.out.println("Error atrapado: " + e.getMessage());
            // "Edad inválida: -5"
        }
    }
}
```

### 🪲 "Rompe el Código"

```java
// ❌ ERROR: setter que no usa this — asigna el parámetro a sí mismo

public class BugSetter {

    private String nombre;
    private int edad;

    public void setNombre(String nombre) {
        nombre = nombre;   // ❌ asigna el PARÁMETRO a sí mismo — el atributo no cambia
        // Sin 'this.', Java no puede distinguir entre el parámetro y el atributo
        // cuando tienen el mismo nombre
    }

    public void setEdad(int edad) {
        this.edad = edad;  // ✅ this.edad (atributo) = edad (parámetro)
    }

    public static void main(String[] args) {
        BugSetter obj = new BugSetter();
        obj.setNombre("Ana");
        System.out.println(obj.nombre);  // null ← el atributo nunca fue asignado

        obj.setEdad(30);
        System.out.println(obj.edad);    // 30 ✅
    }
}

// ✅ REGLA: cuando el parámetro tiene el mismo nombre que el atributo,
// SIEMPRE usar this.atributo = parametro;
```
